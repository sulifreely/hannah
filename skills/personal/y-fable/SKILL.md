# Y Fable

## 核心定位

这个 skill 用一则真正像寓言的短故事，间接解释一个抽象概念。

重点不是把概念翻译成童话，也不是在故事里安排角色讲课，而是让一个具体场景、一两次互动和少量细节自然承载概念结构。读者读完故事时，应先感到“这个场景有意思”，接近结尾才隐约意识到它在讲某个概念。

## 输入判断

如果用户已经给出 `{concept}`，直接写作，不要追问。

如果概念很宽，例如“系统”“自由”“复杂性”，先自行选择一个可解释的具体切面，并在概念解析里说明所取的定义边界。只有当用户的概念完全无法判断含义时，才问一个简短澄清问题。

## 写作流程

### 1. 先在心里拆概念，不要展示过程

动笔前先弄清楚：

- 这个概念属于哪个学科、流派或问题域。
- 它的核心机制是什么，不只是字面定义。
- 它至少包含哪些必要部分：角色、关系、变化、约束、代价或边界。
- 哪些日常场景能把这些部分自然包起来。

不要把这一步输出给用户。故事正文不能出现概念名称，也不能出现该领域术语。

### 2. 选择一个具体场景

优先选择能让概念自己发生的场景：

- 非人类视角：工具、动物、植物、器物、机构、流程中的一个小部件。
- 现代职业和微观场景：理赔、夜班护理、电梯保养、外卖站、菜市场、二手房带看、仓库分拣、门诊排队、客服回访。
- 一个小动作：一次退货、一次维修、一通电话、一次拆卸、一次交接、一张单据被反复修改。

角色不超过三个，最好两个。角色关系本身要承载寓意，而不是靠对白解释。

### 3. 写寓言正文

正文控制在 1000 字以内。用一个核心场景、一两次转折推进。

正文必须遵守：

- 不加标题，不加“下面是一则寓言”之类引导语。
- 不出现概念名称。
- 不使用该概念所属领域的术语。
- 不让角色跳出来解释道理。
- 不在结尾写“这个故事告诉我们……”。
- 让细节承担意义：动作、顺序、误会、代价、残留物、失败方式，比抽象议论更重要。

### 4. 输出概念解析

故事后另起一段，使用 `概念解析` 小标题。

解析要完成三件事：

1. 说明这个概念叫什么，属于哪个流派、学科或问题域，核心定义是什么。
2. 逐一对应故事元素和概念结构：谁映射什么，互动映射什么，转折映射什么，结局映射什么。
3. 说明故事没有覆盖的边界，避免读者把寓言理解成完整理论。

### 5. 输出检验问题

最后使用 `检验问题` 小标题，提出两个具体可答的问题：

1. **理解检验**：检验用户是否理解概念核心机制，而不是只记住故事情节。
2. **迁移检验**：让用户把概念迁移到相关领域，自己举一个例子。

问题要具体，不要写成“你怎么看待这个概念”。

## 反套路自检

动笔前避开这些高频套路。它们会让寓言显得像概念作文，而不是一个有生命的小故事。

### 意象黑名单

不要使用：钟、河流、镜子、迷宫、织布机、地图、灯塔、棋盘、回声、影子、沙漏、风、蜡烛、种子、桥、星辰、蝴蝶、蛛网。

### 地名黑名单

不要写“回声城”“记忆之村”“遗忘之海”“寂静谷”这类过度文艺化虚构地名。给普通地理名词，或者干脆不命名。

### 结构黑名单

不要使用：

- 旅行者求教智者。
- 村庄异象，然后众人顿悟。
- 孩童一句话点醒大人。
- 师徒辩难。
- 临终遗言。

### 角色黑名单

不要使用：钟表匠、图书管理员、隐士、说书人、老船夫、酿酒师、铁匠、抄经人。

### 开头黑名单

不要用“从前有个地方……”“某天某人遇见某事……”“在很远的山里……”这类起手式。直接进入场景。

## 输出格式

严格使用三部分：

```markdown
[寓言正文。无标题、无引导语、1000 字以内。]

## 概念解析

1. [概念名称、所属流派/学科/问题域、核心定义。]
2. [故事元素与概念结构的逐一映射。]
3. [这个寓言的解释边界。]

## 检验问题

1. **理解检验**：[具体问题。]
2. **迁移检验**：[具体问题。]
```

## 质量标准

好的输出应该满足：

- 用户删掉解析部分后，故事仍然像一则完整寓言。
- 用户读解析时，会觉得故事里的细节不是装饰，而是有意设计的映射。
- 故事不是“概念 A 戴上动物头套”，而是用具体关系重演概念机制。
- 解析准确、节制，不因为寓言好看而牺牲概念定义。
- 检验问题能让用户暴露理解盲区。

## 常见错误

| 错误 | 修正 |
| --- | --- |
| 在正文里直接出现概念名或术语 | 换成动作、误会、制度、物件变化来暗示 |
| 角色用对白解释道理 | 让角色做选择并承担后果 |
| 意象太文艺，像散文诗 | 换成现代职业、具体工具、微观交易 |
| 故事过长，铺陈世界观 | 删除背景，只保留一个场景和必要转折 |
| 解析只复述剧情 | 明确写出概念定义和元素映射 |
| 问题太空泛 | 改成能判断对错或能要求具体迁移例子的题 |
