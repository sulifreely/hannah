# 类型 / 风格 / 色板画廊与提示词模板

写提示词（工作流第 4 步）时读取本文件。三个维度自由组合，例如「信息图 × 手绘笔记 × 暖色」。

## 类型（决定信息结构）

| 类型 | 适合 | 画面结构 |
|---|---|---|
| `信息图 infographic` | 数据、指标、技术要点 | 分区块 + 图标 + 少量关键数字 |
| `场景 scene` | 叙事、情绪、代入感 | 一个具体人物/物件在做一件事 |
| `流程图 flowchart` | 过程、步骤、状态流转 | 有向节点 + 箭头，左→右或上→下 |
| `对比 comparison` | 二选一、优劣、新旧 | 左右分屏或上下分栏 |
| `框架 framework` | 模型、架构、分层 | 嵌套/分层/象限结构 |
| `时间线 timeline` | 历史、演进、阶段 | 一条主轴 + 节点标注 |
| `封面 cover` | 文章 hero、记忆点 | 一个强隐喻主体，留标题安全区 |

结构性强、需要精确表达时序或复杂调用链的图，优先退回正文 mermaid/excalidraw，本 skill 只做“看一眼就懂”的示意与插画。

## 风格（决定渲染方式）

| 风格 | 气质 | 默认色板 | 适配类型 |
|---|---|---|---|
| `手绘笔记`（默认） | 白板/漫画感草图线条（rough.js/handDrawn 那种），亲切通用 | 暖色 | 全部，尤其流程/框架/场景 |
| `极简扁平` | 干净的扁平矢量插画感，大色块、细描边 | 默认 | 信息图、对比、框架 |
| `暖色编辑` | 杂志社论插画，具象隐喻、有笔触 | 暖色 | 场景、封面、隐喻图 |
| `蓝图` | 深底 + 细白线技术制图感 | 霓虹/冷色 | 框架、流程、架构 |
| `水彩` | 柔和晕染、留白多、情绪化 | 马卡龙 | 场景、封面、叙事 |

同一篇文章只选**一个**风格，靠它统一系列感；色板可整篇统一，也可整篇换一个。

## 色板（可选，覆盖风格默认色）

| 色板 | 描述 |
|---|---|
| `默认` | 用风格自带的配色 |
| `暖色` | 米白/纸张底 + 赭石、橙、暖棕，柔和亲切 |
| `马卡龙` | 低饱和粉/蓝/绿/黄，柔和教学感 |
| `霓虹` | 深底 + 高饱和荧光强调色，科技/赛博感 |

## 提示词模板

`GenerateImage` 的 `description` 建议按这几段写清楚（中文英文都可，关键是结构齐全）：

```
主体/构图：<画面主体是什么、如何摆放；对应「类型」的信息结构>
关键标签：<从文章里摘的真实文字：数字、术语、短引用；没有就写“无文字”>
风格：<手绘笔记 / 极简扁平 / ...，描述线条、质感、笔触>
色板：<暖色 / 马卡龙 / ...，点明主色 + 强调色>
比例/尺寸：<如 16:9 横图约 1600×900；封面 16:9，信息图可竖 4:5>
避免：<避免的东西：写实照片感、杂乱背景、错误文字、水印等>
```

要点：

- 图上文字越少越稳。确需文字时只放极短标签（1-4 字/词），并在提示词里逐字写出正确内容；大段文字、代码、精确数据一律交给正文。
- 比例默认：封面/横向示意 16:9；信息图/框架可用 4:5 或 1:1；场景按内容定。把它写进 `description`。
- 延续画风：整篇第二张起，把首图路径放进 `reference_image_paths`，并在 `风格`/`色板` 段复述同样的描述词。

### 各类型提示词侧重

- **信息图**：强调分区、图标化、每区一个关键数字或短标签；克制留白，别塞满。
- **场景**：强调一个具体动作与情绪，环境细节服务主题，不写实照片化。
- **流程图**：强调节点顺序与箭头方向，节点内只放极短词；复杂时序改用 mermaid。
- **对比**：强调左右/上下的对称构图与一眼可辨的差异（颜色、体量、方向）。
- **框架**：强调层级/嵌套/象限的空间关系，用位置表达从属，不靠文字解释。
- **时间线**：强调一条清晰主轴 + 均匀节点，阶段用图标或颜色区分。
- **封面**：强调一个能代表全文的隐喻主体，画面上半/一侧留出标题安全区，不在图里写标题文字。
