问答 18（基础）：typeof、keyof、in 操作符 ​

题目 ​

请分别解释 TS 中 typeof、keyof、in 这三个操作符的作用：它们各自用在什么地方？如何和索引访问类型 T[K] 组合使用？能写出哪些常见的工具类型？

参考答案 ​

这三个操作符是 TS 类型编程的"基础三件套"——typeof 从值拿类型、keyof 从对象类型拿键的联合、in 在映射类型里遍历键。加上索引访问类型 T[K]，四者组合可以玩出很多花样。

1. typeof：从值获取类型

在 JS 中，typeof 是运行时操作符，返回一个字符串（'string'、'object' 等）。

在 TS 中，typeof 还有一个类型层面的用法：写在类型位置上时，它会获取一个值的 TS 类型。

ts
const person = { name: 'Alice', age: 30 };

type Person = typeof person;
// { name: string; age: number }

const nums = [1, 2, 3];
type NumArray = typeof nums;  // number[]

常见用途：

  1. 复用已有值的类型：不想重复写类型定义，直接从值上 typeof 一下。
  2. 和 ReturnType / Parameters 配合：从函数值提取参数或返回值类型。
  3. 获取枚举的类型：typeof MyEnum 得到枚举对象的类型。
ts
function greet(name: string) {
  return `Hello, ${name}!`;
}

type GreetFn = typeof greet;       // (name: string) => string
type GreetRet = ReturnType<GreetFn>; // string

注意：typeof 只能作用于值（变量、函数、对象等），不能直接作用于类型。

2. keyof：获取对象类型键的联合

keyof T 接受一个对象类型 T，返回它所有键的联合类型（string | number | symbol 的子类型）。

ts
interface User {
  id: string;
  name: string;
  age: number;
}

type UserKeys = keyof User;
// 'id' | 'name' | 'age'

日常用得更多的是对象的 keyof。

常见用途：

  1. 约束泛型参数：K extends keyof T——K 必须是 T 的键之一。
  2. 和映射类型配合：{ [K in keyof T]: ... }——遍历 T 的所有键。
  3. 和索引访问配合：T[K]——获取键对应的值类型。

3. in：在映射类型中遍历联合

in 用在映射类型（Mapped Types）里，意思是"遍历这个联合类型的每个成员，作为键"。

ts
type Keys = 'a' | 'b' | 'c';

type RecordOfBools = { [K in Keys]: boolean };
// { a: boolean; b: boolean; c: boolean }
1
2
3
4

in 后面跟的是一个键的联合类型，前面的 K 是循环变量（可以随便取名）。

in 和 keyof 经常一起用：

ts
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};
1
2
3

这就是 Partial 的实现——keyof T 拿到键的联合，K in keyof T 逐个遍历，T[K] 拿到对应的值类型。

4. 索引访问类型 T[K]

T[K] 用来获取对象类型 T 中键 K 对应的值的类型。

ts
interface User {
  id: string;
  name: string;
  age: number;
}

type NameType = User['name'];  // string
type AgeType = User['age'];    // number

四者组合的经典案例

这四个东西组合起来，可以实现很多内置工具类型：

Pick<T, K>——选取部分键：

ts
type Pick<T, K extends keyof T> = {
  [P in K]: T[P];
};

type UserBasic = Pick<User, 'id' | 'name'>;
// { id: string; name: string }

追问 1：typeof 和 keyof 能组合用吗？能举个实际例子吗？ ​

答 ​

当然可以！而且非常常用。typeof 先从值拿到类型，keyof 再拿这个类型的键——一步到位。

ts
const config = {
  host: 'localhost',
  port: 3000,
  enabled: true,
};

// 先 typeof 拿类型，再 keyof 拿键
type ConfigKeys = keyof typeof config;
// 'host' | 'port' | 'enabled'

  return config[key];
}

getConfig('host');  // ✅

const colors = {
  red: '#ff0000',
  green: '#00ff00',
  blue: '#0000ff',

追问 2：keyof any 是什么？为什么 Record 的参数用 K extends keyof any？ ​

答 ​

keyof any 是 TS 中"可以作为对象键的类型"的全集——也就是 string | number | symbol。

ts
type KeyOfAny = keyof any;
// string | number | symbol
1
2

为什么 Record 的签名是 Record<K extends keyof any, V>？

因为对象的键只能是 string、number、symbol 这三种（JS 的特性）。用 K extends keyof any 约束 K，确保传进来的 K 是合法的键类型。

对比一下：

ts
// 不好：只约束了 string，不支持 number / symbol 键
type BadRecord<K extends string, V> = { [P in K]: V };

// 好：约束了所有合法键类型
type GoodRecord<K extends keyof any, V> = { [P in K]: V };
1
2
3
4
5

keyof any 本质上是"键类型的上界"——任何能当对象键的类型都是它的子类型。

顺带一提，TS 还有一个内置类型叫 PropertyKey，就是 string | number | symbol 的别名——和 keyof any 等价。

追问 3：T[keyof T] 和 T[K] 有什么区别？keyof T 放在索引访问里有什么用？ ​

答 ​

T[keyof T] 是"所有值类型的联合"——把每个键对应的值类型都拿出来，再联合在一起。

ts
interface User {
  id: string;
  name: string;
  age: number;
}

type UserValues = User[keyof User];
// string | number
// （id → string, name → string, age → number，联合起来就是 string | number）
1
2
3
4
5
6
7
8
9

T[keyof T] 的用途：

  • 获取对象"所有可能的值的类型"
  • 有时候比一个个写联合更方便（特别是字段多的时候）

T[K] 是什么？T[K] 是泛型语境下的写法，K 是一个类型参数，通常有 K extends keyof T 的约束。

ts
function get<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}
1
2
3

这里 T[K] 的意思是"根据传入的具体 K，返回对应的值类型"——调用时 TS 会根据你传的 key 是什么字面量，推断出精确的返回类型。

两者的关系：

  • T[keyof T] 是"所有值类型的联合"，是一个确定的类型
  • T[K] 是"某个键对应的值类型"，依赖 K 是什么，是泛型参数
ts
type All = User[keyof User];      // string | number —— 固定的联合
type One<K extends keyof User> = User[K];  // 依赖 K
type NameType = One<'name'>;      // string —— 具体的类型
1
2
3

一个实用技巧：想知道一个对象"值的联合类型"，直接 T[keyof T]，比自己写 string | number | ... 省事，也不容易漏。
