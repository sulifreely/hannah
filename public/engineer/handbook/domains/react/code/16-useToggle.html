编程题 16（初级）：自定义 Hook：useToggle / useBoolean ​

题目 ​

实现一个 useToggle Hook（也叫 useBoolean），用于管理布尔类型的状态。要求：

  1. 接收一个可选的初始值，默认为 false。
  2. 返回一个元组：[value, toggle, setTrue, setFalse]。
    • value：当前布尔值。
    • toggle：切换值的函数，也可传入一个值直接设置。
    • setTrue：设置为 true。
    • setFalse：设置为 false。
  3. toggle 函数引用保持稳定，不随重渲染变化。
  4. 简洁实用，代码量小但语义清晰。

参考答案 ​

jsx
import { useState, useCallback, useMemo } from 'react';

export function useToggle(initialValue = false) {
  const [value, setValue] = useState(initialValue);

  const toggle = useCallback((nextValue) => {
    if (typeof nextValue === 'boolean') {
      setValue(nextValue);
    } else {
      setValue((v) => !v);
    }
  }, []);

  const setTrue = useCallback(() => setValue(true), []);
  const setFalse = useCallback(() => setValue(false), []);

  // 用 useMemo 保证数组引用稳定（可选优化）
  return useMemo(
    () => [value, toggle, setTrue, setFalse],
    [value, toggle, setTrue, setFalse]
  );
}

// 别名：useBoolean
export const useBoolean = useToggle;

关键点：

  • useCallback：toggle、setTrue、setFalse 都用 useCallback 包裹，确保引用稳定，传给子组件不会破坏 React.memo 优化。
  • 函数式更新：setValue((v) => !v) 使用函数式更新，保证基于最新值取反，避免闭包陷阱。
  • 可选参数：toggle(true) 或 toggle(false) 可直接设置值，也可以不传参直接切换，灵活性高。
  • useMemo 返回数组：如果不加 useMemo，每次渲染返回的都是新数组，解构赋值没问题，但如果把整个数组作为 prop 传递，会破坏引用相等。加上 useMemo 是加分项。
  • 别名 useBoolean：社区里两种命名都常见，useToggle 更强调「切换」动作，useBoolean 更强调类型。

追问 1：toggle 函数什么时候需要传参数？什么时候直接调用 toggle() 更好？ ​

答 ​

两种用法各有适用场景：

直接调用 toggle()（不传参）：

  • 场景：按钮点击切换、开关切换、展开/收起。
  • 特点：「翻转」语义，调用处不关心当前状态，只管切换。
  • 例子：
    jsx
    <button onClick={toggle}>展开/收起</button>
    1
  • 优点：代码简洁，调用方不需要知道当前值。

传参 toggle(nextValue)：

  • 场景：根据外部条件设置明确的值、从 props 派生状态、重置到指定状态。
  • 特点：「设置」语义，调用方明确知道要设置成什么。
  • 例子：
    jsx
    // 外部控制打开
useEffect(() => {
  if (shouldOpen) toggle(true);
}, [shouldOpen]);

// 重置到初始状态
const reset = () => toggle(false);

  • 注意：传参时其实和直接调用 setTrue / setFalse 等价，但 toggle(true) 在动态传入布尔值时更灵活（比如值来自变量）。

使用建议：

  • 如果语义是「切换」→ 用 toggle()
  • 如果语义是「设置为某个确定值」→ 用 setTrue / setFalse 或 toggle(value)
  • 两者都支持是为了灵活性，但项目中最好约定一种惯用写法，保持代码风格一致。

追问 2：如果要支持「函数式初始值」（和 useState 一样支持 useToggle(() => expensiveCompute())），怎么改？ ​

答 ​

useState 本身已经支持惰性初始化，所以只要把参数透传即可。关键点是类型判断——因为我们的 toggle 也支持传函数（函数式更新），所以初始值的函数和更新函数的函数要区分开。

jsx
import { useState, useCallback, useMemo } from 'react';

export function useToggle(initialValue = false) {
  // useState 本身就支持函数式初始值，直接传就行
  const [value, setValue] = useState(initialValue);

  const toggle = useCallback((nextValue) => {
    if (typeof nextValue === 'boolean') {
      setValue(nextValue);
    } else if (typeof nextValue === 'function') {
      // 支持函数式更新：toggle(prev => !prev)
      setValue(nextValue);
    } else {
      // 不传参或传非布尔非函数，取反
      setValue((v) => !v);
    }
  }, []);

  const setTrue = useCallback(() => setValue(true), []);
  const setFalse = useCallback(() => setValue(false), []);

  return useMemo(
    () => [value, toggle, setTrue, setFalse],
    [value, toggle, setTrue, setFalse]
  );
}

注意事项：

  1. useState 惰性初始化：useState(() => compute()) 只在首次渲染执行，之后忽略。所以 useToggle(() => heavyCalc()) 天然支持。
  2. toggle 传函数的歧义：
    • 如果用户写 toggle(() => true)，你怎么知道他是想「设置为 true」还是「取反」？
    • 上面实现选择把函数当成更新函数（和 useState 一致），即 toggle(prev => !prev) 有效。
    • 但要注意：如果传一个返回布尔值的函数，语义是「用函数的返回值作为新值」，不是「切换」。
  3. 实际项目中：大多数场景不需要传函数给 toggle，直接用 setTrue / setFalse 更清晰。函数式更新更多是为了和 useState API 对齐，或者在复杂逻辑中用到。

追问 3：除了布尔值切换，这个模式还能扩展到什么场景？比如枚举值切换、数字自增？ ​

答 ​

useToggle 的核心模式是「状态 + 一组语义化的更新函数」，可以推广到各种简单状态管理场景：

1. 枚举值循环切换（useCycle） ​

jsx
function useCycle(options, initialIndex = 0) {
  const [index, setIndex] = useState(initialIndex);

  const next = useCallback(() => {
    setIndex((i) => (i + 1) % options.length);
  }, [options.length]);

  const prev = useCallback(() => {
    setIndex((i) => (i - 1 + options.length) % options.length);
  }, [options.length]);

  const set = useCallback((val) => {
    const i = options.indexOf(val);
    if (i !== -1) setIndex(i);
  }, [options]);

  return [options[index], { next, prev, set, index }];
}

// 用法
const [theme, { next, prev }] = useCycle(['light', 'dark', 'system'], 0);

2. 计数器（useCounter） ​

jsx
function useCounter(initial = 0, step = 1) {
  const [count, setCount] = useState(initial);
  const inc = useCallback(() => setCount((c) => c + step), [step]);
  const dec = useCallback(() => setCount((c) => c - step), [step]);
  const reset = useCallback(() => setCount(initial), [initial]);
  return [count, { inc, dec, reset, set: setCount }];
}

3. 数组状态（useArray / useList） ​

jsx
function useArray(initial = []) {
  const [list, setList] = useState(initial);
  const push = useCallback((item) => setList((l) => [...l, item]), []);
  const removeAt = useCallback((i) => setList((l) => l.filter((_, idx) => idx !== i)), []);
  const clear = useCallback(() => setList([]), []);
  return [list, { push, removeAt, clear, set: setList }];
}

设计思路 ​

这类「迷你状态 Hook」的共同特点：

  1. 返回 [value, actions] 或 [value, ...actionFns]——和 useState 一致的元组风格，好用好记。
  2. actions 引用稳定——都用 useCallback 包裹，配合 memo 子组件不破坏优化。
  3. 语义化命名——inc/dec/push/clear 比 setValue(...) 更具可读性。
  4. 保留原始 setter——高级用法仍然可以绕过封装直接操作。

什么时候该封装？

  • 同一套状态 + 操作在多处重复出现 → 封装成 Hook。
  • 只有一处用、逻辑又简单 → 直接 useState 就行，不必为了封装而封装。
