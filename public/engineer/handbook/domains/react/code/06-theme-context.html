编程题 06（中级）：主题切换（Context + localStorage） ​

题目 ​

实现明暗主题切换：

  • 使用 Context 提供 { theme, toggle }；
  • 页面刷新后保持上次主题；
  • 支持系统偏好（prefers-color-scheme）作为初始值；
  • 避免不必要的重渲染（拆分 Context 或 memo）。

参考答案 ​

jsx
import { createContext, useContext, useEffect, useMemo, useState, useCallback } from "react";

const ThemeCtx = createContext(null);

function getInitial() {
  const saved = localStorage.getItem("theme");
  if (saved === "light" || saved === "dark") return saved;
  return window.matchMedia("(prefers-color-scheme: dark)").matches ? "dark" : "light";
}

export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState(getInitial);

  useEffect(() => {
    document.documentElement.dataset.theme = theme;
    localStorage.setItem("theme", theme);
  }, [theme]);

  const toggle = useCallback(
    () => setTheme((t) => (t === "light" ? "dark" : "light")),
    []
  );

  const value = useMemo(() => ({ theme, toggle }), [theme, toggle]);
  return <ThemeCtx.Provider value={value}>{children}</ThemeCtx.Provider>;
}

export function useTheme() {
  const ctx = useContext(ThemeCtx);
  if (!ctx) throw new Error("useTheme must be used inside ThemeProvider");
  return ctx;
}

关键点：

  • 惰性初始化 useState(getInitial) 只在首次渲染读取 localStorage；
  • useMemo 稳定 value 引用，避免所有消费者被无谓触发；
  • toggle 使用 useCallback + 函数式更新，无需在依赖里放 theme。

追问 ​

追问 1：为什么把 value 用 useMemo 包起来？不加会有什么问题？ ​

参考答案：Provider 每次渲染会新建 { theme, toggle } 对象，即使字段不变，引用也不同。所有消费该 Context 的组件都会因为 value 引用变化而重渲染。用 useMemo 缓存对象，让引用只在 theme/toggle 真的变化时更新。

追问 2：如何让"只用 toggle"的组件不因 theme 变化而重渲染？ ​

参考答案：把 setter/theme 拆分到两个 Context：ThemeStateCtx（值）+ ThemeActionCtx（动作）。只订阅动作的组件不会被值的变化触发。或者引入 use-context-selector 让消费者按字段订阅。

追问 3：SSR（服务端渲染）时如何避免首屏"闪一下"错误主题？ ​

参考答案：服务端拿不到 localStorage 和 matchMedia，需要采取 Hydration 前的手动脚本：在 <head> 里内联一段脚本，读取 cookie/localStorage 并把 data-theme 写到 <html> 上，浏览器解析 HTML 时就已应用正确主题，避免 Hydration 后闪烁。Next.js 里可用 next-themes 之类的库。
