问答 08（进阶）：动态规划入门与常见题型 ​

题目 ​

请讲清楚动态规划的核心思想（最优子结构、重叠子问题、无后效性）、解题五步方法论，并通过经典题型（爬楼梯、不同路径、最小路径和、01 背包、完全背包、最长递增子序列、最长公共子序列）举例说明。

参考答案 ​

一、动态规划核心思想 ​

动态规划（Dynamic Programming，DP）是通过把大问题拆成小问题、保存子问题的解避免重复计算来求解最优化问题的方法。

三要素 / 三个性质 ​

  1. 最优子结构（Optimal Substructure）

    • 大问题的最优解可以由子问题的最优解构造出来。
    • 例：爬楼梯到第 n 阶的方法数 = 到第 n-1 阶的方法数 + 到第 n-2 阶的方法数。

  2. 重叠子问题（Overlapping Subproblems）

    • 递归求解时会反复计算相同的子问题。
    • 这是「存子问题解」的理由——如果子问题都不重叠，直接分治就行，不需要 DP。

  3. 无后效性

    • 一旦确定了某个子问题的解，就不再受更大问题的决策影响。
    • 换句话说：「过去不依赖未来」，状态只由前面的状态转移而来，不考虑后面怎么走。
    • 有后效性的问题通常需要增加状态维度来消除后效性。

DP vs 递归 / 分治 vs 贪心

方法特点适用
递归 / 分治子问题独立，不重叠子问题不重复（如归并排序）
DP子问题重叠 + 最优子结构 + 无后效性最优化 / 计数问题
贪心每步选局部最优，期望得到全局最优需要满足贪心选择性质

二、解题五步方法论 ​

Carl 哥的五步法，非常好用：

  1. 确定 dp 数组及下标的含义：dp[i] 代表什么？维度是几维的？
  2. 确定递推公式（状态转移方程）：dp[i] 怎么从前面的状态推出来？
  3. dp 数组如何初始化：边界条件是什么？
  4. 确定遍历顺序：从前到后？从后到前？两层循环谁在外谁在内？
  5. 举例推导 dp 数组：手算几步验证递推是否正确。

为什么要按这五步来？ 因为 DP 最容易出错的地方就是「递推公式想错了」或「初始化错了」或「遍历顺序反了」。按步走可以系统排查，面试时也能展示清晰的思路。

三、经典题型解析 ​

1. 爬楼梯（LeetCode 70 - 入门） ​

题目：每次可以爬 1 或 2 个台阶，爬到第 n 阶有多少种方法？

五步走：

  1. dp[i]：爬到第 i 阶的方法数。
  2. 递推：dp[i] = dp[i-1] + dp[i-2]（最后一步是 1 阶或 2 阶）。
  3. 初始化：dp[1] = 1，dp[2] = 2。
  4. 遍历：从前到后（3 → n）。
  5. 举例：n=5 → 1, 2, 3, 5, 8 = 斐波那契。
js
function climbStairs(n) {
  if (n <= 2) return n;
  let a = 1, b = 2;
  for (let i = 3; i <= n; i++) {
    const c = a + b;
    a = b; b = c;
  }
  return b;
}

2. 不同路径（LeetCode 62 - 二维入门） ​

题目：m × n 网格，从左上到右下，只能向右或向下走，有多少条不同路径？

五步走：

  1. dp[i][j]：走到 (i, j) 的路径数。
  2. 递推：dp[i][j] = dp[i-1][j] + dp[i][j-1]（从上来或从左来）。
  3. 初始化：第一行 dp[0][j] = 1，第一列 dp[i][0] = 1（只有一种走法）。
  4. 遍历：从上到下、从左到右。
  5. 举例：3×3 → [[1,1,1],[1,2,3],[1,3,6]]。
js
function uniquePaths(m, n) {
  const dp = Array.from({ length: m }, () => new Array(n).fill(1));
  for (let i = 1; i < m; i++)
    for (let j = 1; j < n; j++)
      dp[i][j] = dp[i-1][j] + dp[i][j-1];
  return dp[m-1][n-1];
}

3. 最小路径和（LeetCode 64 - 带权值） ​

题目：m × n 网格，每个格子有非负数，找一条从左上到右下的路径，使路径上数字和最小。

递推：dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])。

和不同路径的区别：一个是「求和（计数）」，一个是「取 min（最优化）」。模板几乎一样。

4. 01 背包（经典中的经典） ​

题目：有 n 件物品，第 i 件重量 w[i]、价值 v[i]，背包最多装重量 W。每件物品只能选一次，求最大价值。

五步走：

  1. dp[i][j]：从前 i 件物品中选，放入容量为 j 的背包，最大价值。
  2. 递推：dp[i][j] = max(dp[i-1][j], dp[i-1][j - w[i]] + v[i])（不选第 i 件 / 选第 i 件）。
  3. 初始化：dp[0][j] = 0（没有物品价值为 0），dp[i][0] = 0（容量为 0 价值为 0）。
  4. 遍历：物品在外层，容量在内层（二维无所谓顺序）。
  5. 举例：略。
js
// 二维 DP
function knapsack01(W, weights, values) {
  const n = weights.length;
  const dp = Array.from({ length: n + 1 }, () => new Array(W + 1).fill(0));
  for (let i = 1; i <= n; i++) {
    for (let j = 1; j <= W; j++) {
      if (j < weights[i-1]) dp[i][j] = dp[i-1][j];
      else dp[i][j] = Math.max(dp[i-1][j], dp[i-1][j - weights[i-1]] + values[i-1]);
    }
  }
  return dp[n][W];
}

function knapsack01(W, weights, values) {
  const dp = new Array(W + 1).fill(0);
  for (let i = 0; i < weights.length; i++) {
    // 注意：倒序遍历容量！防止同一件物品被多次放入

      dp[j] = Math.max(dp[j], dp[j - weights[i]] + values[i]);

  }
  return dp[W];
}

5. 完全背包 ​

题目：每件物品可以选无限次，其他同 01 背包。

递推：把 01 背包一维版的容量遍历改成正序就可以了。

js
function knapsackComplete(W, weights, values) {
  const dp = new Array(W + 1).fill(0);
  for (let i = 0; i < weights.length; i++) {
    for (let j = weights[i]; j <= W; j++) {  // 正序
      dp[j] = Math.max(dp[j], dp[j - weights[i]] + values[i]);
    }
  }
  return dp[W];
}

  • 求组合数（零钱兑换 II）：物品在外层、容量在内层 → 组合数。
  • 求排列数（爬楼梯进阶、组合总和 IV）：容量在外层、物品在内层 → 排列数。
  • 面试常问「组合 vs 排列」的遍历顺序区别，要能讲清。

6. 最长递增子序列（LIS，LeetCode 300） ​

题目：给一个整数数组，找出最长严格递增子序列的长度。（子序列不要求连续）

DP 解法（O(n²)）：

  1. dp[i]：以 nums[i] 结尾的最长递增子序列长度。
  2. 递推：dp[i] = max(dp[j] + 1)，其中 j < i 且 nums[j] < nums[i]。
  3. 初始化：每个元素至少为 1。
  4. 遍历：i 从 0 到 n-1，每个 i 内遍历 j 从 0 到 i-1。
  5. 结果：max(dp[i])。
js
function lengthOfLIS(nums) {
  const n = nums.length;
  const dp = new Array(n).fill(1);
  let res = 1;
  for (let i = 1; i < n; i++) {
    for (let j = 0; j < i; j++) {
      if (nums[j] < nums[i]) dp[i] = Math.max(dp[i], dp[j] + 1);
    }
    res = Math.max(res, dp[i]);
  }
  return res;
}

7. 最长公共子序列（LCS，LeetCode 1143） ​

题目：两个字符串 text1 和 text2，返回它们的最长公共子序列长度。

DP 解法（O(nm)）：

  1. dp[i][j]：text1 前 i 个字符和 text2 前 j 个字符的 LCS 长度。
  2. 递推：
    • 如果 text1[i-1] === text2[j-1]：dp[i][j] = dp[i-1][j-1] + 1（两个字符相同，LCS 加 1）
    • 否则：dp[i][j] = max(dp[i-1][j], dp[i][j-1])（去掉 text1 的第 i 个或去掉 text2 的第 j 个，取大的）
  3. 初始化：dp[0][j] = 0，dp[i][0] = 0。
  4. 遍历：两个字符串从左到右。
js
function longestCommonSubsequence(text1, text2) {
  const m = text1.length, n = text2.length;
  const dp = Array.from({ length: m + 1 }, () => new Array(n + 1).fill(0));
  for (let i = 1; i <= m; i++) {
    for (let j = 1; j <= n; j++) {
      if (text1[i-1] === text2[j-1]) dp[i][j] = dp[i-1][j-1] + 1;
      else dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
    }
  }
  return dp[m][n];
}

追问 1：01 背包一维优化为什么要倒序遍历容量？如果正序会发生什么？请用具体数值举例说明。 ​

答 ​

核心原因：一维 DP 是在「复用同一个数组」模拟二维 DP 的「上一行」。倒序是为了保证 dp[j - w[i]] 取到的是「上一层（i-1）」的值，而不是「本层（i）」已经更新过的值。

具体举例：假设只有一件物品，重量 3，价值 5，背包容量 6。

正确做法：倒序（6 → 3）

初始 dp = [0, 0, 0, 0, 0, 0, 0]

i=0（物品0，w=3, v=5）：
j=6: dp[6] = max(0, dp[3] + 5) = max(0, 0+5) = 5  → dp[3] 还是上一层的 0
j=5: dp[5] = max(0, dp[2] + 5) = 0
j=4: dp[4] = max(0, dp[1] + 5) = 0
j=3: dp[3] = max(0, dp[0] + 5) = 5

结果 dp = [0, 0, 0, 5, 0, 0, 5]
含义：容量 3 和 6 都能放这件物品，最大价值 5。（01 背包只能放一次，容量 6 放了一件还剩 3 但没别的物品了）

为什么 j=6 错了？因为正序时 dp[3] 在 j=3 时已经被更新成了 5（本层的值），dp[6 - 3] = dp[3] = 5 表示「在已经放了一件物品 0 的基础上再放一件」，这就把同一件物品放了两次——这是完全背包的逻辑，不是 01 背包。

一句话总结：

  • 倒序 → 用上一层的旧值 → 每件物品只用一次 → 01 背包
  • 正序 → 用本层的新值 → 每件物品可用无限次 → 完全背包

这个区别是背包问题最核心的考点，面试几乎必考。

追问 2：「无后效性」到底是什么意思？能不能举一个「有后效性」的例子，并说明怎么通过增加状态维度消除后效性？ ​

答 ​

无后效性的定义：

一旦确定了 dp[i] 的值，它就只和 i 之前的状态有关，不会被 i 之后的决策所改变。换句话说，「过去的状态」不依赖「未来的选择」。

更正式地说：第 i 个状态的取值，只由前 i-1 个状态的取值和第 i 步的决策决定；一旦算出了 dp[i]，不管后面怎么选，dp[i] 都不会变。

反例：有后效性的问题

例子：「粉刷房子」的一个变种。

正常版（无后效性）：一排 n 个房子，每个房子可以刷成红/蓝/绿三种颜色，相邻颜色不能相同，每个颜色有不同花费。求最小总花费。

  • dp[i][c]：刷完前 i 个房子，第 i 个刷颜色 c 的最小花费。
  • 递推：dp[i][0] = min(dp[i-1][1], dp[i-1][2]) + cost[i][0]
  • 无后效性：前 i 个房子的状态确定后，不影响后面怎么刷。

加条件变成有后效性：规定「整个序列中蓝色不能使用超过 k 次」。

这时如果你只定义 dp[i][c] 就不够了——因为「蓝色还能用几次」会影响后面的决策，但当前状态里没有记录这个信息。dp[i][红色] 可能对应「已经用了 2 次蓝色」也可能对应「已经用了 0 次蓝色」，这两种状态后续的可选方案完全不同。

这就是有后效性：当前状态无法独立存在，它的「前途」依赖于没有记录在状态里的历史信息。

消除后效性：增加状态维度

把「已使用蓝色的次数」也放进状态里：

  • dp[i][c][k]：前 i 个房子，第 i 个刷颜色 c，已经用了 k 次蓝色时的最小花费。

或者简化（只在刷蓝色时 k 才增加）：

  • 当 c ≠ 蓝色时：dp[i][c][k] = min(dp[i-1][其他色][k]) + cost[i][c]
  • 当 c = 蓝色时：dp[i][蓝][k] = min(dp[i-1][其他色][k-1]) + cost[i][蓝]

这样状态里包含了所有影响未来决策的信息，就消除了后效性。

常见的「加维度消后效」场景：

  • 「最多选 k 次」→ 加一个选了几次的维度
  • 「买股票带冷冻期 / 手续费」→ 加一个当前是否持有股票的维度（如 dp[i][0/1] 表示第 i 天不持有/持有）
  • 「状态转移依赖上两步」→ 加一个上一步状态的维度
  • 「区间 DP」→ 天然是二维的，因为区间端点都要记录

判断有无后效性的方法：问自己「知道了当前 dp 状态的值，能不能直接推下一步？还是说还需要别的历史信息？」如果还需要别的信息，就说明状态定义得不够，需要加维度。

追问 3：LIS 的 O(n log n) 贪心+二分算法是什么原理？为什么「维护最小末尾」能得到最长长度？ ​

答 ​

算法回顾：

维护一个数组 tails，其中 tails[i] 表示「长度为 i+1 的递增子序列的最小末尾元素值」。

遍历数组 nums 中每个数 x：

  • 如果 x > tails 最后一个元素 → 直接追加到 tails 末尾（找到了更长的递增子序列）。
  • 否则 → 用二分查找找到 tails 中第一个 ≥ x 的位置，用 x 替换它（让这个长度的子序列末尾更小，为后续数字留出更大空间）。

最终 tails 的长度就是 LIS 的长度。

js
function lengthOfLIS(nums) {
  const tails = [];
  for (const x of nums) {
    // 二分找 tails 中第一个 >= x 的位置
    let l = 0, r = tails.length;
    while (l < r) {
      const mid = (l + r) >> 1;
      if (tails[mid] < x) l = mid + 1;
      else r = mid;
    }
    if (l === tails.length) tails.push(x);
    else tails[l] = x;
  }
  return tails.length;
}

为什么这样是对的？「最小末尾」和「最长长度」有什么关系？

核心直觉：对于同样长度的递增子序列，末尾越小，后面能接的数字就越多，越有利于延长序列。

举个具体例子：nums = [10, 9, 2, 5, 3, 7, 101, 18]

处理的数tails 变化说明
10[10]第一个数，长度 1 的子序列最小末尾是 10
9[9]9 < 10，替换。长度 1 的末尾变成 9（更小了，后面更有优势）
2[2]继续替换，长度 1 的末尾是 2
5[2, 5]5 > 2，追加。长度 2 的子序列最小末尾是 5（对应子序列 [2,5]）
3[2, 3]3 < 5，替换位置 1。长度 2 的末尾从 5 变成 3（更小了）
7[2, 3, 7]7 > 3，追加
101[2, 3, 7, 101]追加
18[2, 3, 7, 18]替换 101 为 18

最终 tails 长度 = 4，正确答案就是 4（如 [2,3,7,18] 或 [2,3,7,101]）。

关键理解：

  1. tails 数组本身一定是严格递增的（反证法：如果 tails[i] >= tails[i+1]，说明长度为 i+2 的子序列的最小末尾 ≤ 长度 i+1 的子序列末尾，但长度 i+2 的子序列的第 i+1 个元素肯定比末尾小，那它就应该是 tails[i] 的更优解，矛盾）。正因为 tails 递增，所以可以用二分查找。

  2. tails 的长度就是 LIS 的长度，但 tails 本身不一定是真正的 LIS 序列——它存的是「各长度下的最小末尾」，不是完整的子序列。如果要输出具体的 LIS，需要额外记录每个元素对应的长度等信息，最后回溯。

  3. 为什么替换是安全的？ 替换 tails 中某个位置的值，不会改变 tails 的长度，也不会破坏 tails 的递增性。它只是让「某个长度的子序列末尾更小」，这是一个「不亏」的操作——如果未来有数字能接在旧值后面，那它一定也能接在更小的新值后面；反过来则不一定。所以替换操作要么维持现状，要么为未来创造更多可能。

这个算法的精妙之处在于：它不直接求「最长子序列是什么」，而是维护一个「各个长度的最优结尾」的结构，用贪心策略不断优化这个结构，最终结构的规模就是答案。理解了这个「维护最优边界」的思想，对掌握很多高级算法都有启发。
