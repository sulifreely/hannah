function levelOrder(root) {
  if (!root) return [];

    const levelSize = queue.length;

2. 二叉搜索树（BST） ​

性质：

  • 左子树所有节点 < 根 < 右子树所有节点
  • 中序遍历得到递增序列（BST 最核心的特征）
  • 平均查找 / 插入 / 删除 O(log n)，最坏 O(n)（退化成链表）

验证 BST：中序遍历后检查是否严格递增，或递归时传递上下界。

js
function isValidBST(root, min = -Infinity, max = Infinity) {
  if (!root) return true;
  if (root.val <= min || root.val >= max) return false;
  return isValidBST(root.left, min, root.val)
      && isValidBST(root.right, root.val, max);
}

3. 平衡二叉树 ​

BST 的问题：最坏退化成链表。平衡树通过旋转操作保持高度在 O(log n)。

AVL 树 ​

  • 严格平衡：每个节点左右子树高度差（平衡因子）不超过 1。
  • 通过左旋 / 右旋 / 左右旋 / 右左旋四种旋转维持平衡。
  • 查找快（严格平衡），但插入删除旋转多，写性能差。
  • 适合读多写少的场景。

红黑树 ​

  • 近似平衡：通过五条红黑性质保证最长路径 ≤ 2 × 最短路径。
    1. 节点非红即黑
    2. 根节点黑
    3. 叶子（NIL）黑
    4. 红节点的孩子必须黑（不能有连续红）
    5. 任意节点到其所有叶子的路径上黑节点数相同（黑高相同）
  • 插入删除最多 2 次 / 3 次旋转，写性能优于 AVL。
  • 工程中广泛使用：Java TreeMap、C++ map、Linux 进程调度。
对比项AVL红黑树
平衡严格度严格（高度差 ≤1）近似（最长 ≤ 2×最短）
查找更快（更平衡）略慢

4. 堆（Heap） ​

定义：完全二叉树 + 堆序性质（父 ≥ 子 = 大顶堆，父 ≤ 子 = 小顶堆）。

数组表示：完全二叉树天然用数组存，索引 i 的节点：

  • 左孩子：2i + 1
  • 右孩子：2i + 2
  • 父节点：(i - 1) >> 1

核心操作：

  • 上浮（sift up / bubble up）：插入时用，O(log n)。
  • 下沉（sift down / heapify）：删除堆顶时用，O(log n)。
  • 建堆：从最后一个非叶子节点往前逐个下沉，O(n)（不是 O(n log n)！因为叶子多的层操作少，总和等比数列求和是 O(n)）。

JS 实现小顶堆

js
class MinHeap {
  constructor() { this.heap = []; }

  push(val) {
    this.heap.push(val);
    this._siftUp(this.heap.length - 1);
  }

  pop() {
    if (this.heap.length === 0) return null;
    const top = this.heap[0];
    this.heap[0] = this.heap.pop();
    this._siftDown(0);
    return top;
  }

  peek() { return this.heap[0] ?? null; }

  _siftUp(i) {
    while (i > 0) {
      const p = (i - 1) >> 1;
      if (this.heap[p] > this.heap[i]) {
        [this.heap[p], this.heap[i]] = [this.heap[i], this.heap[p]];
        i = p;
      } else break;
    }
  }

  _siftDown(i) {
    const n = this.heap.length;
    while (true) {
      let smallest = i;
      const l = 2 * i + 1, r = 2 * i + 2;
      if (l < n && this.heap[l] < this.heap[smallest]) smallest = l;
      if (r < n && this.heap[r] < this.heap[smallest]) smallest = r;
      if (smallest !== i) {
        [this.heap[i], this.heap[smallest]] = [this.heap[smallest], this.heap[i]];
        i = smallest;
      } else break;
    }
  }
}

TopK 问题：求数组中第 K 大的元素。

  • 解法一：小顶堆存前 K 大，遍历数组比堆顶大就替换，O(n log K)，空间 O(K)。适合海量数据流（不用把数据全加载进来）。
  • 解法二：快排思想（快速选择 QuickSelect），平均 O(n)，最坏 O(n²)，原地 O(1)。适合内存装得下的数据。

5. 图的表示 ​

邻接矩阵 ​

  • 二维数组 matrix[i][j] 表示 i 到 j 是否有边 / 权重。
  • 检查边 O(1)，但空间 O(V²)，稀疏图浪费大。
  • 适合稠密图或需要快速查边的场景。

邻接表 ​

  • 每个节点维护一个列表（链表 / 数组 / Set），存它的邻居。
  • 空间 O(V+E)，稀疏图高效。
  • 遍历邻居方便，查边要遍历列表 O(degree)。

DFS vs BFS 怎么选？

  • 找最短路径（无权图）→ BFS（层序遍历，第一次到达就是最短）。
  • 找所有路径 / 连通性 / 环检测→ DFS 更直观，代码短。
  • 图很深（递归栈可能爆）→ BFS 或迭代 DFS。
  • 图很宽（BFS 队列可能爆内存）→ DFS。

7. 最短路径：Dijkstra 算法 ​

适用：带权图（权重非负）的单源最短路径。

思想：贪心 + 优先队列。每次选当前距离最短的未访问节点，用它「松弛」邻居的距离。

js
function dijkstra(graph, start) {
  const dist = {};
  for (const node of graph.keys()) dist[node] = Infinity;
  dist[start] = 0;

  // 简化：用数组 + sort 模拟优先队列，实际应该用堆
  const pq = [[0, start]]; // [distance, node]

  while (pq.length) {
    pq.sort((a, b) => a[0] - b[0]);
    const [d, u] = pq.shift();
    if (d > dist[u]) continue;  // 已经找到更短路径，跳过

    for (const [v, w] of graph.get(u) || []) {
      if (dist[u] + w < dist[v]) {
        dist[v] = dist[u] + w;       // 松弛操作
        pq.push([dist[v], v]);
      }
    }
  }
  return dist;
}

追问 1：为什么堆排序是 O(n log n) 但建堆只要 O(n)？建堆为什么从后往前而不是从前往后？ ​

答 ​

建堆为什么是 O(n) 而不是 O(n log n)？

建堆（heapify）是从最后一个非叶子节点开始，向前遍历每个节点并执行下沉操作。直觉上觉得每个节点 O(log n) 共 n 个节点 → O(n log n)，但实际上越靠上层的节点越少，下沉的深度越大；越靠下层的节点越多，下沉的深度越小。

精确计算（假设是满二叉树，高度 h = log₂n）：

  • 第 0 层（根）：1 个节点，最多下沉 h 步
  • 第 1 层：2 个节点，最多下沉 h-1 步
  • 第 2 层：4 个节点，最多下沉 h-2 步
  • ...
  • 第 h-1 层（倒数第二层）：2^(h-1) 个节点，最多下沉 1 步
  • 第 h 层（叶子）：2^h 个节点，下沉 0 步

总操作量：

S = Σ(k=1 to h) [2^(h-k) × k]
  = 2^h × Σ(k=1 to h) [k / 2^k]
1
2

其中 Σ(k=1 to ∞) k/2^k = 2（等比数列导数公式可证），所以 S ≤ 2^h × 2 = 2^(h+1) ≈ 2n。

因此建堆是 O(n)，不是 O(n log n)。

为什么从后往前（下沉）而不是从前往后（上浮）？

如果从前往后建堆（逐个插入 + 上浮），那复杂度就是 O(n log n)：

  • 第 k 个元素插入，最多上浮 log₂k 步
  • 总和 Σ(k=1 to n) log k ≈ n log n

而从后往前（逐个下沉），如上面分析是 O(n)。

根本原因：叶子节点占一半（n/2），它们不需要下沉；而上浮法每个节点都可能一路浮到顶。下沉法「大头（叶子）不用动，小头（根）多动」，总代价更小。

这也解释了为什么堆排序整体是 O(n log n)：建堆 O(n) + n 次删除堆顶（每次 O(log n)）= O(n + n log n) = O(n log n)。

追问 2：BST 中序遍历为什么是有序的？能用这个性质解决哪些经典问题？ ​

答 ​

为什么中序遍历有序？

BST 的定义是：左子树所有节点 < 根 < 右子树所有节点。

中序遍历顺序是「左 → 根 → 右」。对于任意节点，先遍历完所有比它小的左子树，再访问它自己，最后遍历所有比它大的右子树。递归地看，每个节点都把自己放在左（小）和右（大）之间，所以整个序列自然是递增的。

这是 BST 最核心的性质，也是面试最高频的考点。

能解决的经典问题：

  1. 验证 BST（LeetCode 98）：中序遍历后检查是否严格递增。O(n) 时间。

  2. BST 中第 K 小的元素（LeetCode 230）：中序遍历到第 K 个就返回。O(h + K)，h 是树高。

  3. BST 中的众数（LeetCode 501）：中序遍历相同的数连续出现，用变量追踪当前数的出现次数和最大值。

  4. 把 BST 转换成累加树 / Greater Tree（LeetCode 538）：反向中序（右→根→左）遍历，累加前缀和。

  5. 两数之和 IV - 输入 BST（LeetCode 653）：中序得到有序数组，再双指针。

  6. BST 的最小绝对差（LeetCode 530）：中序遍历，相邻元素差的最小值。

  7. 二叉搜索树迭代器（LeetCode 173）：用栈模拟中序遍历的迭代版，实现 hasNext() 和 next()。

进阶技巧：

  • 中序 + 前驱节点（prev）的模式非常通用，很多 BST 题都能用「维护一个 prev 指针 + 中序遍历」搞定。
  • 如果要从大到小遍历，就用「右→根→左」的反向中序。
  • Morris 中序遍历可以做到 O(1) 额外空间（利用叶子的空右指针做线索），面试进阶考点。

追问 3：Dijkstra 为什么不能处理负权边？Bellman-Ford 和 Floyd-Warshall 分别适用于什么场景？ ​

答 ​

Dijkstra 为什么不能处理负权边？

Dijkstra 的核心假设是：一旦一个节点被「弹出优先队列」（即标记为已访问），它的最短距离就确定了，不会再被更新。这是基于「所有边权非负」的贪心选择性质——你不可能通过绕更多的路得到更短的路径（因为每加一条边距离只会增加或不变）。

如果有负权边，这个假设就不成立了。举个反例：

A --1--> B --1--> C
|                 ^
└------- -5------┘
1
2
3

从 A 出发：

  1. Dijkstra 先选 A（dist=0），更新 B=1，C=-5。
  2. 优先队列里 C(-5) < B(1)，弹出 C，标记为已访问。
  3. 然后弹出 B，更新 C = min(-5, 1+1=-3) = -5，没变。

但实际上 A→B→C 的距离是 2，而 A→C 是 -5……等等，这个例子好像 Dijkstra 还对了。让我换一个经典反例：

A --2--> B
|        |
5        -10
|        |
v        v
C --1--> D

从 A 出发到 D：

  • Dijkstra：先弹 A，更新 B=2、C=5。弹 B，更新 D = 2 + (-10) = -8。弹 C，更新 D = min(-8, 5+1=-4) = -8。结果对。

S --5--> A
|        |
1        1
|        |

B ---(-3)--> C
|        ^
└---1----┘

实际上 S→A→... 让我们简化：核心问题是 Dijkstra 一旦标记节点为「已确定」就不再更新，但负权边可能通过后来的路径让一个「已确定」的节点距离变得更短。

简单说：Dijkstra 的贪心策略依赖「边权非负 ⇒ 先出队的就是最短」。有负权就不成立了。

Bellman-Ford 适用场景：

  • 有负权边的单源最短路径。
  • 能检测负权环（如果第 n 轮还能松弛，说明有负权环，不存在最短路径）。
  • 时间复杂度 O(V·E)，比 Dijkstra 高。
  • 适合图比较小或者有负权的情况。

Floyd-Warshall 适用场景：

  • 多源最短路径（求任意两点间最短距离）。
  • 动态规划思想：d[i][j] = min(d[i][j], d[i][k] + d[k][j])，k 是中间节点。
  • 时间 O(V³)，空间 O(V²)。
  • 代码极简（三重循环），适合节点数少的图（如 V ≤ 500）。
  • 可以处理负权边，但不能有负权环（和 Bellman-Ford 一样能检测）。

总结选型：

算法适用时间负权多源
Dijkstra非负权单源O((V+E)logV)❌❌
Bellman-Ford有负权单源 / 检测负环O(V·E)✅❌
Floyd-Warshall任两点最短路 / 节点少O(V³)✅✅
BFS无权图单源O(V+E)—❌
