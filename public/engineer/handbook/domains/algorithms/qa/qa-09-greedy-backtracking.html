问答 09（进阶）：贪心算法与回溯算法 ​

题目 ​

请分别讲解贪心算法和回溯算法：

  • 贪心：适用条件（贪心选择性质、最优子结构）、经典题（分发饼干、最大子数组和、买卖股票最佳时机、跳跃游戏、柠檬水找零）；
  • 回溯：模板（确定参数 / 终止条件 / 单层搜索逻辑）、经典题（组合、排列、子集、棋盘问题）、剪枝优化。

参考答案 ​

一、贪心算法 ​

核心思想 ​

每一步都选当前看起来最好的选择，期望通过一系列局部最优选择得到全局最优解。

两个适用条件 ​

  1. 最优子结构：大问题的最优解包含子问题的最优解。（和 DP 一样）
  2. 贪心选择性质：全局最优可以通过一系列局部最优选择达到。也就是说，每一步选当下最优的，不需要回头改。（这是贪心比 DP 要求更高的地方）

贪心 vs DP：

  • 贪心是「每步选最好的，不回头」→ 更高效，但适用面窄。
  • DP 是「枚举所有可能，择优而选」→ 适用面广，但复杂度更高。
  • 判断一个题能不能用贪心，关键是能不能证明贪心选择性质（通常靠反证法 / 数学归纳法）。

经典贪心题 ​

1. 分发饼干（LeetCode 455 - 入门） ​

题目：每个孩子有一个胃口值 g[i]，每块饼干有一个尺寸 s[j]。如果 s[j] >= g[i]，这块饼干就能满足这个孩子。每个孩子最多一块饼干。问最多能满足几个孩子？

思路：小饼干先给胃口小的孩子，大饼干留给胃口大的。双指针遍历。

js
function findContentChildren(g, s) {
  g.sort((a, b) => a - b);
  s.sort((a, b) => a - b);
  let i = 0, j = 0;
  while (i < g.length && j < s.length) {
    if (s[j] >= g[i]) { i++; j++; }  // 能满足，下一个孩子
    else j++;                          // 不够大，换更大的饼干
  }
  return i;
}

2. 最大子数组和（LeetCode 53 - Kadane 算法） ​

题目：找一个数组中和最大的连续子数组，返回最大和。

js
function maxSubArray(nums) {
  let maxSum = nums[0];
  let curSum = nums[0];
  for (let i = 1; i < nums.length; i++) {
    curSum = Math.max(nums[i], curSum + nums[i]);  // 要么接在前面，要么从自己开始
    maxSum = Math.max(maxSum, curSum);
  }
  return maxSum;
}

3. 买卖股票的最佳时机（LeetCode 121） ​

题目：只能交易一次（买一次卖一次），求最大利润。

js
function maxProfit(prices) {
  let minPrice = prices[0];
  let maxProfit = 0;
  for (let i = 1; i < prices.length; i++) {
    maxProfit = Math.max(maxProfit, prices[i] - minPrice);
    minPrice = Math.min(minPrice, prices[i]);
  }
  return maxProfit;
}

4. 跳跃游戏（LeetCode 55） ​

题目：非负整数数组，每个元素表示在该位置可以跳跃的最大长度。判断能不能到达最后一个位置。

js
function canJump(nums) {
  let cover = 0;  // 当前能覆盖的最远位置
  for (let i = 0; i <= cover; i++) {
    cover = Math.max(cover, i + nums[i]);
    if (cover >= nums.length - 1) return true;
  }
  return false;
}

5. 柠檬水找零（LeetCode 860） ​

题目：顾客买 5 元柠檬水，支付 5/10/20 美元。你一开始没有零钱。判断能不能正确找零。

思路：

  • 收到 5 元 → 直接收下。
  • 收到 10 元 → 找 5 元。
  • 收到 20 元 → 优先找 10+5（因为 10 元只能用来找 20 元的零，留着没用；5 元更灵活），没有 10 元就找 3 张 5 元。
js
function lemonadeChange(bills) {
  let five = 0, ten = 0;
  for (const bill of bills) {
    if (bill === 5) five++;
    else if (bill === 10) {
      if (five === 0) return false;
      five--; ten++;
    } else { // 20
      if (ten > 0 && five > 0) { ten--; five--; }  // 贪心：优先用 10 元
      else if (five >= 3) five -= 3;
      else return false;
    }
  }
  return true;
}

二、回溯算法 ​

核心思想 ​

回溯 = 深度优先搜索 + 撤销选择。

把问题看成一棵「决策树」：每个节点是当前状态，每条边是一个选择。沿着树深度遍历，走到叶子节点（满足终止条件）就记录答案，然后回退一步尝试另一个选择。

回溯本质是暴力枚举 + 剪枝。虽然是暴力，但通过剪枝可以跳过很多不可能的路径。

回溯模板三部曲 ​

  1. 确定函数参数：当前走到哪了（startIndex）、路径（path）、结果集（result）、各种约束条件。
  2. 确定终止条件：什么时候算找到一个解？收集结果后 return。
  3. 单层搜索逻辑：遍历所有可选分支，做选择 → 递归 → 撤销选择（回溯）。
js
function backtrack(参数) {
  if (终止条件) {
    result.push([...path]);  // 收集结果
    return;
  }
  for (选择 of 当前层可选的选择) {
    path.push(选择);           // 做选择
    backtrack(新参数);         // 递归
    path.pop();                // 撤销选择（回溯）
  }
}

经典回溯题 ​

1. 组合（LeetCode 77） ​

题目：给定 n 和 k，返回 1...n 中所有可能的 k 个数的组合。

js
function combine(n, k) {
  const result = [], path = [];
  function backtrack(start) {
    if (path.length === k) {          // 终止条件
      result.push([...path]);
      return;
    }
    // 剪枝：剩余需要选 k - path.length 个，当前 i 最多到 n - (k - path.length) + 1
    for (let i = start; i <= n - (k - path.length) + 1; i++) {
      path.push(i);                    // 选
      backtrack(i + 1);                // 递归，下一层从 i+1 开始（避免重复）
      path.pop();                      // 撤销
    }
  }
  backtrack(1);
  return result;
}

2. 排列（LeetCode 46） ​

题目：给定一个不含重复数字的数组，返回所有可能的全排列。

js
function permute(nums) {
  const result = [], path = [];
  const used = new Array(nums.length).fill(false);

  function backtrack() {
    if (path.length === nums.length) {
      result.push([...path]);
      return;
    }
    for (let i = 0; i < nums.length; i++) {
      if (used[i]) continue;            // 用过的跳过
      used[i] = true;
      path.push(nums[i]);
      backtrack();
      path.pop();
      used[i] = false;                   // 回溯
    }
  }
  backtrack();
  return result;
}

3. 子集（LeetCode 78） ​

题目：给一个整数数组，返回所有可能的子集（幂集）。

js
function subsets(nums) {
  const result = [], path = [];
  function backtrack(start) {
    result.push([...path]);          // 每个节点都是一个子集，进来先收集
    for (let i = start; i < nums.length; i++) {
      path.push(nums[i]);
      backtrack(i + 1);
      path.pop();
    }
  }
  backtrack(0);
  return result;
}

4. N 皇后（LeetCode 51 - 棋盘问题） ​

题目：n × n 棋盘上放 n 个皇后，使它们不能互相攻击（同一行/列/对角线都不行）。返回所有解。

js
function solveNQueens(n) {
  const result = [];
  const board = Array.from({ length: n }, () => new Array(n).fill('.'));

  function isValid(row, col) {
    // 检查列（行不用查，因为每层只放一个）
    for (let i = 0; i < row; i++)
      if (board[i][col] === 'Q') return false;
    // 检查左上对角线
    for (let i = row - 1, j = col - 1; i >= 0 && j >= 0; i--, j--)
      if (board[i][j] === 'Q') return false;
    // 检查右上对角线
    for (let i = row - 1, j = col + 1; i >= 0 && j < n; i--, j++)
      if (board[i][j] === 'Q') return false;
    return true;
  }

  function backtrack(row) {
    if (row === n) {
      result.push(board.map(r => r.join('')));
      return;
    }
    for (let col = 0; col < n; col++) {
      if (!isValid(row, col)) continue;  // 剪枝
      board[row][col] = 'Q';
      backtrack(row + 1);
      board[row][col] = '.';             // 回溯
    }
  }
  backtrack(0);
  return result;
}

棋盘问题特点：二维空间，每行/每列放一个，用行号做递归深度，列数做循环宽度。合法性检查做剪枝。

剪枝优化 ​

回溯是暴力算法，剪枝是灵魂。常见剪枝思路：

  1. 起点剪枝：组合问题的 startIndex，避免重复枚举。
  2. 数量剪枝：剩余数量不够时提前终止（如组合题的 n - (k - path.length) + 1）。
  3. 合法性剪枝：提前判断当前选择是否合法，不合法直接跳过（如 N 皇后的 isValid）。
  4. 求和剪枝：累加和超过目标时跳过（组合总和类题）。
  5. 排序去重剪枝：数组有重复元素时，先排序，同一层中相同元素只选第一个（i > start && nums[i] === nums[i-1] 就跳过）。

三、贪心 vs 回溯 vs DP 对比 ​

算法思想适用场景时间复杂度
贪心每步选局部最优，不回头满足贪心选择性质一般 O(n) 或 O(n log n)
回溯暴力枚举 + 撤销 + 剪枝组合/排列/子集/棋盘等穷举类指数级，但剪枝后实际很快
DP记录子问题解，避免重复计算最优子结构 + 重叠子问题多项式级（O(n) ~ O(n²) 等）

追问 1：什么时候该用贪心而不是 DP？如何证明一道题可以用贪心？ ​

答 ​

用贪心而不用 DP 的前提：题目满足贪心选择性质——即「每一步做最优选择，最终能得到全局最优」。DP 不需要这个性质，因为 DP 会枚举所有可能然后择优。

怎么判断能不能用贪心？ 两个层面：

1. 直觉试探法（面试常用）

先想一个贪心策略，然后找反例。如果找了半天找不到反例，可能就可以用贪心。如果找到了反例，说明贪心不行，得上 DP。

比如：

  • 「零钱兑换」（用最少硬币凑出金额）：如果硬币是 [1, 5, 10, 20]，贪心（每次选最大的）是对的。但如果硬币是 [1, 3, 4]，目标金额是 6——贪心选 4+1+1（3 枚），而最优是 3+3（2 枚）。反例存在，贪心不行。
  • 「活动选择」：每次选结束最早的活动 → 能选最多活动。这个贪心是对的，找不到反例。

2. 严格证明法（进阶要求）

两种常用证明方法：

方法一：反证法

假设存在一个最优解，它的第一步选择和贪心选择不一样。证明可以把最优解的第一步替换成贪心选择，得到的解不会更差 → 贪心选择至少和最优解一样好。

以活动选择为例：

  • 设最优解选择的第一个活动是 A（结束时间 t1），贪心选的是 B（结束时间 t0 ≤ t1）。
  • 把最优解中的 A 换成 B，因为 B 结束更早，后面的活动选择空间更大，得到的解活动数不会比原来少。
  • 所以贪心选择 B 至少和最优解一样好，贪心选择性质成立。

方法二：数学归纳法

证明「前 k 步用贪心选择得到的解，在第 k+1 步用贪心选择仍然最优」。

工程实践中的建议：

  1. 先想贪心：如果贪心策略很直觉（如「先满足最小的」「每次选最大的」「优先用掉最没用的」），先试试。
  2. 疯狂找反例：想几个极端 case 测试。找不到反例基本就对了。
  3. 找到了反例 → 上 DP：DP 是更通用的武器，适用面广但代价高。
  4. 面试时说清楚：即使直接写了贪心，也要提一句「我判断这道题满足贪心选择性质，理由是……」，体现思维的严谨性。

常见的能用贪心的场景关键词：

  • 「最多 / 最少」+ 有明确的「优先级排序」
  • 「每次选最大 / 最小的」
  • 区间调度类（开始结束时间的安排）
  • 霍夫曼编码（最短编码长度）
  • Prim / Kruskal 最小生成树

追问 2：回溯算法的时间复杂度怎么分析？为什么有时候回溯会超时，剪枝到底能剪多少？ ​

答 ​

回溯的时间复杂度 = 决策树的节点数 × 每个节点的处理时间。

因为回溯本质是 DFS 遍历一棵决策树，时间复杂度和树的节点数成正比。

不同题型的节点数：

题型节点数（量级）说明
子集O(2ⁿ)每个元素可选可不选，共 2ⁿ 个子集
组合 C(n, k)O(C(n, k))约等于 n^k / k!，k 固定时是多项式
排列O(n!)n 个元素全排列 n! 种
N 皇后O(n!)第一行 n 种，第二行 n-2 种……约 n!
组合总和（可重复选）O(n^(target/min))指数级，底数是候选数

每个节点的处理时间：通常是 O(1) 或 O(n)（比如收集结果时需要拷贝数组）。

所以整体复杂度：

  • 子集：O(n × 2ⁿ)（2ⁿ 个节点，每个拷贝 O(n)）
  • 排列：O(n × n!)（n! 个节点，每个拷贝 O(n)）
  • N 皇后：O(n × n!)（还要加上合法性检查，实际更快因为剪枝多）

为什么有时候回溯会超时？

  1. n 太大：比如 n=30 的子集问题，2³⁰ ≈ 10 亿，肯定超时。
  2. 剪枝不够：暴力枚举了很多不可能的路径。
  3. 状态重复：同一个状态被反复计算（这种情况应该用 DP / 记忆化搜索）。

剪枝到底能剪多少？

剪枝的效果高度依赖数据，没有统一答案。但可以分几类来看：

1. 起点剪枝（startIndex）

  • 效果：把排列（n!）降为组合（C(n, k)），或者把重复枚举完全干掉。
  • 这是结构性剪枝，改变问题的复杂度量级。

2. 数量剪枝（剩余不够就停）

  • 效果：比如组合问题 i <= n - (k - path.length) + 1，当 path 已经很长时，内层循环提前结束。
  • 大约减少常数因子，不改变量级。

3. 合法性剪枝（isValid 过滤）

  • 效果：因题而异。N 皇后问题中，合法性剪枝非常强——n=8 时有 8! = 40320 种排列，但实际解只有 92 个，大量路径在很早的层就被剪掉了。
  • 数据好的时候可能剪 90%+，数据差的时候可能几乎不剪。

4. 排序去重剪枝

  • 效果：如果有大量重复元素，能把指数级砍掉一大截。比如 [1,1,1,1,1] 求子集，不剪枝会有 2⁵=32 个节点，剪枝后只有 6 个（空集、一个 1、两个 1……五个 1）。

优化回溯性能的手段：

  1. 剪枝！剪枝！剪枝！ 能剪的都剪了。
  2. 排序 + 剪枝：很多题排序后可以提前终止循环（如组合总和，当前和 + nums[i] > target 就 break）。
  3. 用 index 而不是数组切片：传 startIndex 比 slice 数组高效得多。
  4. 避免频繁拷贝：用 push/pop 维护同一个 path，收集结果时才拷贝。
  5. 数据结构加速合法性检查：如 N 皇后可用三个布尔数组（列、左斜、右斜）代替遍历检查。
  6. 如果还是超时 → 换 DP：如果问题有大量重叠子问题，记忆化搜索 / DP 可能是多项式级，比回溯的指数级强得多。

一句话总结：回溯是指数级算法，剪枝只能剪常数或部分分支，不能改变指数级本质。数据规模大的时候（n > 20）就要慎重，该用 DP 就用 DP。

追问 3：回溯和 DFS 是什么关系？记忆化搜索和 DP 又是什么关系？这四者能串起来讲吗？ ​

答 ​

这四个概念经常一起出现，但层次不同。让我从「暴力」到「聪明」一层层串起来：

暴力枚举 → DFS → 回溯 → 记忆化搜索(DP)
1

第一层：暴力枚举（Brute Force）

最朴素的思想：把所有可能的情况都列出来，然后找答案。

实现方式有很多种：

  • 多重 for 循环（维度固定且少的时候）
  • DFS / BFS 遍历所有状态
  • 递归生成所有情况

第二层：DFS（深度优先搜索）

DFS 是一种遍历顺序——一条路走到黑，撞了南墙再回头。它是实现暴力枚举的一种方式。

  • 用递归或栈实现。
  • 特点：空间占用小（只存当前路径），适合找所有解 / 判断可达性。
  • BFS 是另一种遍历顺序，层序遍历，适合找最短路径。

第三层：回溯（Backtracking）

回溯 = DFS + 撤销选择 + 剪枝。

为什么要单独叫「回溯」而不直接叫 DFS？因为回溯有一个典型的模式：

  • 做选择 → 递归 → 撤销选择

这个「撤销选择」的动作就是「回溯」一词的来源——回到上一步，尝试另一个选择。

回溯强调的是在搜索过程中维护状态（path、used 等），进入递归前修改状态，退出递归后恢复状态。

纯 DFS 不一定有这个特征（比如图的 DFS 遍历只是标记 visited，不需要撤销）。

所以关系是：

  • 回溯一定是 DFS 的一种应用
  • 但 DFS 不一定是回溯（比如图的遍历、树的前序遍历是 DFS，但不叫回溯）

第四层：记忆化搜索（Memoization） = 自顶向下 DP

当回溯/DFS 中出现了大量重复的子问题时，每次都重新算一遍很浪费。把算过的结果存起来（缓存），下次遇到直接用——这就是记忆化搜索。

例子：爬楼梯。

纯递归（无记忆化）：

js
function f(n) {
  if (n <= 2) return n;
  return f(n-1) + f(n-2);  // 大量重复计算，O(2ⁿ)
}

记忆化搜索：

js
const memo = {};
function f(n) {
  if (n <= 2) return n;
  if (memo[n]) return memo[n];
  memo[n] = f(n-1) + f(n-2);  // 算过的存起来，O(n)
  return memo[n];
}

记忆化搜索本质就是自顶向下的动态规划——从目标问题出发，递归地拆成子问题，遇到算过的直接取结果。

记忆化搜索 vs 迭代 DP

对比项记忆化搜索（自顶向下）迭代 DP（自底向上）
方向从目标到 base case从 base case 到目标
实现递归 + memo 数组循环 + dp 数组
优点思路自然，状态转移清晰效率高，无递归开销
缺点递归栈开销，可能栈溢出遍历顺序需要想清楚
适用状态转移复杂、不是所有子问题都要算经典 DP 题型、状态规整

二者时间复杂度理论上相同（都算 O(状态数 × 转移数)），只是计算顺序不同。

四者的完整关系链：

暴力枚举
  └── DFS（深度优先的枚举方式）
        ├── 纯 DFS（如图遍历、树遍历，不需要撤销状态）
        └── 回溯（DFS + 撤销选择 + 剪枝，用于组合/排列/子集等搜索问题）
              └── 如果有大量重叠子问题 → 记忆化搜索 = 自顶向下 DP
                    └── 等价于 → 迭代 DP（自底向上）

怎么选用什么？

  1. 题目是「求所有解 / 判断是否存在 / 求组合排列」→ 回溯（搜索 + 剪枝）。
  2. 题目是「求最值 / 求方案数」且有重叠子问题 → DP（记忆化搜索或迭代）。
  3. 数据量小（n ≤ 20）→ 回溯也能过，写起来快。
  4. 数据量大（n ≥ 1000）→ 必须 DP，回溯肯定超时。
  5. 状态转移关系复杂、不容易想遍历顺序 → 先用记忆化搜索写，再考虑转迭代。

理解了这条从「笨」到「聪明」的进化链，就理解了为什么有这么多看似不同实则相通的算法名称。
