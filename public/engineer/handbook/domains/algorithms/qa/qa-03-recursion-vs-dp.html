问答 03（进阶）：递归、迭代、记忆化搜索与动态规划的关系 ​

题目 ​

递归、迭代、记忆化搜索、动态规划这四个概念之间是什么关系？为什么很多 DP 题可以写成「记忆化递归」也可以写成「递推」？什么时候必须用哪种写法？

参考答案 ​

四个概念的定义

  • 递归（recursion）：函数直接或间接调用自己，把大问题拆成同构的小问题。
  • 迭代（iteration）：用循环重复执行一段代码，逐步逼近答案。
  • 记忆化搜索（memoization）：给递归加缓存，相同参数只算一次，避免重复子问题。也叫「自顶向下 DP」。
  • 动态规划（DP）：把问题的解表达为状态，通过状态转移方程从已知状态推出未知状态，通常用循环递推。也叫「自底向上 DP」。

核心关系图

       递归 ──加缓存──> 记忆化搜索 (自顶向下 DP)
        │                    │
        │                    │ 拓扑等价
        ↓                    ↓
       迭代  ──状态+转移──> 动态规划 (自底向上 DP)

为什么可以互相转换 DP 的本质是有向无环图（DAG）上的最短路 / 最长路 / 计数：

  • 每个状态是一个节点；
  • 转移方程是节点间的边；
  • 求解就是按拓扑序遍历这个 DAG。

只要 DAG 结构不变：

  • 从终点递归求解 + 记忆化 = 自顶向下；
  • 从起点按拓扑序（通常是循环递推）计算 = 自底向上。 两者时间复杂度相同（都是「状态数 × 单次转移代价」），只是执行顺序不同。

斐波那契四种写法对比

js
// 1. 纯递归 O(2^n) 指数爆炸
function fib1(n) {
  if (n < 2) return n;
  return fib1(n - 1) + fib1(n - 2);
}

// 2. 记忆化递归 O(n) 时间 O(n) 空间（含栈）
function fib2(n, memo = {}) {
  if (n < 2) return n;
  if (memo[n] != null) return memo[n];
  return memo[n] = fib2(n - 1, memo) + fib2(n - 2, memo);
}

// 3. DP 递推 O(n) 时间 O(n) 空间
function fib3(n) {
  const dp = [0, 1];
  for (let i = 2; i <= n; i++) dp[i] = dp[i - 1] + dp[i - 2];
  return dp[n];
}

// 4. DP + 滚动变量 O(n) 时间 O(1) 空间
function fib4(n) {
  let a = 0, b = 1;
  for (let i = 2; i <= n; i++) [a, b] = [b, a + b];
  return n === 0 ? 0 : b;
}

如何在两种写法之间选择

情境推荐写法
状态维度多、转移方向不清晰记忆化递归（写法直观）
状态图是树 / 图上的 DP（不容易定义拓扑序）记忆化递归
状态维度低、转移方向单一（如线性 DP）迭代 DP，方便滚动数组降空间
需要追踪路径 / 回溯方案迭代 DP 通常更方便
面试时想快速写完记忆化递归上手最快
数据量大、递归深会爆栈迭代 DP
空间要压到 O(1) / O(k)迭代 DP（滚动数组）

必须用递归的场景

  • 树形 DP、状态不易线性化（如树上背包）。
  • 状态转移依赖顺序无法预先确定（如博弈搜索 + 剪枝、DFS + 记忆化）。

必须用迭代的场景

  • JS 中递归深度超过约 1 万会爆栈（V8 默认限制），大数据规模的线性 DP 一定用迭代。
  • 需要滚动数组等极致空间优化。
  • 需要在计算过程中同时构造答案路径。

追问 1：什么样的问题适合用 DP？如何判断？ ​

答：满足两个性质即可考虑 DP：

  • 最优子结构：大问题的最优解可以由小问题的最优解合成。
  • 重叠子问题：递归展开会重复计算相同的子问题。 如果只有最优子结构、没有重叠子问题（例如二叉树独立子树上的独立最优），一般用分治或贪心就够了，不需要 DP。 判断步骤：
  1. 定义状态：能唯一描述子问题的最少信息。
  2. 写出状态转移方程：状态之间的依赖关系。
  3. 确定边界（初始） 和 目标状态。
  4. 判断拓扑序：哪个状态先算，哪个后算。 如果这些都能写清楚，几乎一定能实现。

追问 2：为什么很多 DP 题可以用「滚动数组」把空间从 O(n²) 降到 O(n)？举个例子。 ​

答：

  • 观察状态转移方程：当前层的状态只依赖前 k 层时，就不必保存全部历史层，只保留最近 k 层即可。
  • 最经典的例子是 0-1 背包：dp[i][j] = max(dp[i-1][j], dp[i-1][j-w] + v)，第 i 行只依赖第 i-1 行，可以只用一维数组，且必须逆序遍历 j 保证每个物品只用一次。
  • 类似地，最长公共子序列（LCS）也可以从 O(mn) 空间降到 O(min(m, n))，通过滚动数组 + 只保留两行。
  • 这个技巧的关键是弄清楚状态依赖了哪几层、遍历顺序如何保证不覆盖将来还要用的数据。

追问 3：记忆化搜索遇到「爆栈」怎么办？ ​

答：几种解决路径：

  1. 改成迭代 DP：按拓扑序循环递推，从根本上避免深递归。这是最推荐的做法。
  2. 手动模拟栈：把递归改成显式栈 + 状态机，本质就是把递归调用栈搬到堆上。适合难以直接找到拓扑序的场景。
  3. 尾递归改写：某些语言支持 TCO，可以自动展开成循环。JS 大部分引擎不做，仅参考。
  4. 增大栈大小：Node 可以用 --stack-size 调整，但只是权宜之计，治标不治本。
  5. 蹦床（trampoline）：把每次递归调用改成返回一个「thunk」，外层循环反复调用，把递归展开成迭代。写法稍绕，但完全消除爆栈风险，通用性强。
