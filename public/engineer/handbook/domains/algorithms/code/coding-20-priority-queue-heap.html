编程 20（进阶）：优先队列（堆）实现 ​

题目 ​

用 JavaScript 实现一个优先队列（Priority Queue）类，底层用二叉堆（Binary Heap）实现。

要求：

  1. 支持小顶堆（默认）和大顶堆两种模式（通过 comparator 自定义比较）
  2. 实现以下方法：
    • push(val)：插入元素，O(log n)
    • pop()：弹出堆顶元素（最小值或最大值），O(log n)
    • peek()：查看堆顶元素，O(1)
    • size()：返回堆的大小，O(1)
    • isEmpty()：是否为空，O(1)
  3. 基于数组存储，实现上浮（sift up）和下沉（sift down）操作
  4. 支持从已有数组构建堆（heapify），O(n) 时间

参考答案 ​

完整实现 ​

js
class PriorityQueue {
  constructor(comparator = (a, b) => a - b) {
    // comparator(a, b) < 0 表示 a 排在 b 前面（优先级更高）
    // 默认 a - b：小顶堆（小的在前）
    this.heap = [];
    this.compare = comparator;
  }

  // 堆大小
  size() {
    return this.heap.length;
  }

  // 是否为空
  isEmpty() {
    return this.heap.length === 0;
  }

  // 查看堆顶
  peek() {
    return this.heap[0];
  }

  // 插入元素
  push(val) {
    this.heap.push(val);          // 先放到数组末尾
    this._siftUp(this.heap.length - 1); // 然后上浮到正确位置
  }

  // 弹出堆顶
  pop() {
    if (this.heap.length === 0) return undefined;
    const top = this.heap[0];     // 保存堆顶
    const last = this.heap.pop(); // 把最后一个元素移到堆顶
    if (this.heap.length > 0) {
      this.heap[0] = last;
      this._siftDown(0);          // 然后下沉到正确位置
    }
    return top;
  }

  // 上浮：从 index 开始向上比较，如果比父节点优先级高就交换
  _siftUp(index) {
    while (index > 0) {
      const parent = (index - 1) >> 1; // 父节点索引：(i-1)/2 向下取整
      // 如果当前节点优先级比父节点高（compare < 0），交换
      if (this.compare(this.heap[index], this.heap[parent]) < 0) {
        [this.heap[index], this.heap[parent]] = [this.heap[parent], this.heap[index]];
        index = parent;
      } else {
        break; // 已经满足堆性质，停止
      }
    }
  }

  // 下沉：从 index 开始向下比较，和优先级更高的子节点交换
  _siftDown(index) {
    const n = this.heap.length;
    while (true) {
      let smallest = index; // 假设当前节点优先级最高
      const left = 2 * index + 1;  // 左子节点索引
      const right = 2 * index + 2; // 右子节点索引
      
      // 和左子节点比
      if (left < n && this.compare(this.heap[left], this.heap[smallest]) < 0) {
        smallest = left;
      }
      // 和右子节点比
      if (right < n && this.compare(this.heap[right], this.heap[smallest]) < 0) {
        smallest = right;
      }
      
      if (smallest !== index) {
        // 和优先级更高的子节点交换
        [this.heap[index], this.heap[smallest]] = [this.heap[smallest], this.heap[index]];
        index = smallest;
      } else {
        break; // 已经满足堆性质，停止
      }
    }
  }

  // 从已有数组构建堆（heapify），O(n) 时间
  heapify(arr) {
    this.heap = arr.slice(); // 复制一份
    // 从最后一个非叶子节点开始，从后往前依次下沉
    // 最后一个非叶子节点的索引：Math.floor(n/2) - 1 = (n >> 1) - 1
    for (let i = (this.heap.length >> 1) - 1; i >= 0; i--) {
      this._siftDown(i);
    }
  }
}

核心概念讲解 ​

1. 为什么用数组存堆？ ​

二叉堆是一棵完全二叉树，所以可以用数组紧凑地存储，不需要左右孩子指针：

数组索引：0    1    2    3    4    5    6
堆结构： 根
        /   \
       1     2
      / \   / \
     3   4 5   6
1

3
4

6

父子节点索引关系：

  • 父节点：parent(i) = (i - 1) >> 1（即 Math.floor((i-1)/2)）
  • 左子节点：left(i) = 2*i + 1
  • 右子节点：right(i) = 2*i + 2

也有从索引 1 开始存的写法（父节点 = i/2，左孩子 = 2i，右孩子 = 2i+1），但 JS 数组从 0 开始，用 0 起点更自然。

2. 上浮（Sift Up / Bubble Up） ​

插入新元素时：

  1. 把新元素放到数组末尾（树的最后一个位置）
  2. 不断和父节点比较，如果优先级比父节点高就交换位置
  3. 直到父节点优先级更高，或者到达根节点

时间 O(log n)：最多交换树高层数。

3. 下沉（Sift Down / Bubble Down） ​

删除堆顶时：

  1. 把最后一个元素移到堆顶
  2. 不断和两个子节点比较，和优先级更高的那个子节点交换
  3. 直到两个子节点优先级都更低，或者到达叶子节点

时间 O(log n)：最多交换树高层数。

4. Heapify（建堆）为什么是 O(n) 而不是 O(n log n)？ ​

直觉上好像每个节点下沉 O(log n)，n 个节点就是 O(n log n)，但实际上：

  • 叶子节点（约 n/2 个）根本不用下沉
  • 倒数第二层（约 n/4 个）最多下沉 1 层
  • 倒数第三层（约 n/8 个）最多下沉 2 层
  • ...
  • 根节点（1 个）下沉 log n 层

总操作数 = 0 × n/2 + 1 × n/4 + 2 × n/8 + ... + log n × 1 = O(n)

所以从后往前、从最后一个非叶子节点开始下沉的建堆方式是 O(n) 时间，比一个个 push 的 O(n log n) 更快。

追问 1：Top K 问题（数组中第 K 大的元素）怎么用堆实现？时间复杂度是多少？为什么用小顶堆而不是大顶堆？ ​

答 ​

题目：在未排序的数组中找到第 k 个最大的元素。例如 [3,2,1,5,6,4], k=2 → 5。

方法一：小顶堆（维护前 K 大的元素，推荐） ​

js
function findKthLargest(nums, k) {
  const minHeap = new PriorityQueue(); // 小顶堆
  
  for (const num of nums) {
    minHeap.push(num);
    // 如果堆大小超过 k，弹出最小的（保持堆里是当前最大的 k 个）
    if (minHeap.size() > k) {
      minHeap.pop();
    }
  }
  
  return minHeap.peek(); // 堆顶就是第 k 大的元素
}

思路：

  • 维护一个大小为 k 的小顶堆，堆中存「当前遍历过的元素中最大的 k 个」。
  • 遍历数组，每个元素都加入堆。
  • 如果堆的大小超过 k，就把堆顶（最小的那个）弹出去——因为它肯定不是前 k 大的。
  • 遍历完后，堆顶就是「第 k 大」的元素（因为堆是小顶堆，堆顶是这 k 个里最小的，也就是全局第 k 大）。

复杂度：

  • 时间 O(n log k)：每个元素入堆出堆一次，堆大小 k，所以是 n × log k。
  • 空间 O(k)：堆的大小。

为什么用小顶堆而不是大顶堆？

  • 小顶堆的堆顶是「前 k 大」中最小的那个——也就是第 k 大。我们要找的就是它。
  • 如果用大顶堆维护前 k 大，堆顶是第 1 大的，我们没法快速得到第 k 大的。
  • 反过来，如果要找「第 k 小」的元素，就用大顶堆（堆顶是前 k 小中最大的 = 第 k 小）。

记忆技巧：找第 K 大用小顶堆（堆顶就是答案），找第 K 小用大顶堆。

方法二：大顶堆（全部元素建堆，弹 k 次） ​

js
function findKthLargest2(nums, k) {
  const maxHeap = new PriorityQueue((a, b) => b - a); // 大顶堆
  maxHeap.heapify(nums); // O(n) 建堆
  
  let result;
  for (let i = 0; i < k; i++) {
    result = maxHeap.pop(); // 弹 k 次
  }
  return result;
}

拓展：前 K 个高频元素 ​

js
function topKFrequent(nums, k) {
  // 1. 统计频率
  const freq = new Map();
  for (const num of nums) {
    freq.set(num, (freq.get(num) || 0) + 1);
  }
  
  // 2. 小顶堆：按频率排序，维护频率最高的 k 个
  const minHeap = new PriorityQueue((a, b) => a[1] - b[1]); // [num, 频率]
  
  for (const [num, count] of freq) {
    minHeap.push([num, count]);
    if (minHeap.size() > k) {
      minHeap.pop();
    }
  }
  
  // 3. 收集结果
  const result = [];
  while (!minHeap.isEmpty()) {
    result.push(minHeap.pop()[0]);
  }
  return result.reverse(); // 从高到低排列
}

追问 2：堆排序（Heap Sort）怎么实现？它的时间空间复杂度是多少？和快速排序比有什么优缺点？ ​

答 ​

堆排序实现 ​

思路：

  1. 把数组构建成大顶堆（O(n)）
  2. 每次把堆顶（最大值）和数组末尾交换，然后堆大小减 1，对新的堆顶下沉
  3. 重复 n-1 次，数组就从小到大排好了
js
function heapSort(arr) {
  const n = arr.length;
  
  // 1. 建大顶堆
  for (let i = (n >> 1) - 1; i >= 0; i--) {
    siftDown(arr, n, i);
  }
  
  // 2. 一个个把堆顶移到末尾
  for (let i = n - 1; i > 0; i--) {
    [arr[0], arr[i]] = [arr[i], arr[0]]; // 堆顶和末尾交换
    siftDown(arr, i, 0); // 对新堆顶下沉（堆大小是 i）
  }
  
  return arr;
}

// 大顶堆的下沉操作
function siftDown(arr, size, index) {
  while (true) {
    let largest = index;
    const left = 2 * index + 1;
    const right = 2 * index + 2;
    
    if (left < size && arr[left] > arr[largest]) largest = left;
    if (right < size && arr[right] > arr[largest]) largest = right;
    
    if (largest !== index) {
      [arr[index], arr[largest]] = [arr[largest], arr[index]];
      index = largest;
    } else {
      break;
    }
  }
}

复杂度：

  • 时间 O(n log n)：建堆 O(n) + n 次下沉 O(n log n) = O(n log n)。
  • 空间 O(1)：原地排序，只用了常数额外空间。
  • 不稳定排序：相等元素的相对顺序可能改变。

和快速排序对比 ​

维度堆排序快速排序
平均时间O(n log n)O(n log n)
最坏时间O(n log n)O(n²)（但随机化后几乎不会出现）
空间O(1) 原地O(log n) 递归栈
稳定性不稳定不稳定
缓存友好性较差（跳着访问）较好（顺序访问分区）
实际速度较慢较快（常数项小）

为什么实际中快排比堆排快？

  1. 缓存局部性：快排的分区操作是顺序访问数组的，堆排的下沉操作是跳来跳去的（2i+1），缓存命中率低。
  2. 常数项：堆排每次交换后都要继续下沉，比较和交换的次数更多。
  3. 实际场景：大多数编程语言的内置排序（如 V8 的 Array.sort）在数据量大时用 TimSort（归并 + 插入排序的混合）或 QuickSort，而不是堆排。

堆排序的优势：

  • 最坏情况也是 O(n log n)（快排最坏 O(n²)，虽然可以随机化避免）
  • 完全原地，空间 O(1)（归并排序需要 O(n) 额外空间）
  • 适合「需要维护动态集合的最值」的场景（优先队列），而不是纯排序

追问 3：除了二叉堆，你还知道哪些堆的数据结构？它们各有什么优势？ ​

答 ​

二叉堆是最基础、最常用的堆，但还有其他更高级的堆结构，在某些操作上有更好的性能：

1. 二项堆（Binomial Heap） ​

  • 特点：由若干棵二项树组成（森林结构）。
  • 优势：支持**两个堆的合并（merge）**在 O(log n) 时间完成。二叉堆的合并是 O(n) 的。
  • 其他操作：插入、删除最值、减小关键字都是 O(log n) 均摊。
  • 应用：需要频繁合并优先队列的场景。

2. 斐波那契堆（Fibonacci Heap） ​

  • 特点：理论上非常优美的数据结构，森林结构 + 级联剪枝。
  • 优势：
    • 插入 O(1) 均摊
    • 取最小值 O(1)
    • 减小关键字（decrease key）O(1) 均摊
    • 合并 O(1) 均摊
  • 劣势：删除最小值仍是 O(log n) 均摊；实际实现非常复杂，常数项很大，实测不一定比二叉堆快。
  • 应用：Dijkstra 算法、Prim 算法中用斐波那契堆可以把时间从 O(m log n) 降到 O(m + n log n)（理论上）。

3. 左式堆（Leftist Heap）/ 斜堆（Skew Heap） ​

  • 特点：也是支持高效合并的堆结构。
  • 左式堆：保证左子树的「零距离」比右子树大，合并 O(log n)。
  • 斜堆：左式堆的简化版，不维护秩，每次合并都交换左右子树，均摊 O(log n)。
  • 优势：实现比二项堆、斐波那契堆简单，合并效率高。

4. 配对堆（Pairing Heap） ​

  • 特点：结构简单，实际性能优秀。
  • 优势：decrease key 操作很快（实际接近 O(1)，但理论分析还没证明）。
  • 应用：很多实际库（如 GCC 的 C++ 扩展 __gnu_pbds::priority_queue）有配对堆实现。

5. 索引堆（Index Heap） ​

  • 特点：在普通二叉堆基础上加了「反向索引」，可以快速找到任意元素在堆中的位置。
  • 优势：支持「修改指定元素的值」操作 O(log n)。普通堆要找一个元素得 O(n) 遍历。
  • 应用：Dijkstra 算法中需要「更新某节点的距离」，用索引堆更高效。

实际选择建议：

  • 99% 的场景：二叉堆就够了，实现简单、性能够用。
  • 需要合并堆：考虑二项堆或左式堆。
  • 面试中：掌握二叉堆的实现和原理是必须的，知道有更高级的堆结构（斐波那契堆等）及其优势即可，一般不会要求手写。

补充：JavaScript 中直到 ES202x 都没有内置的优先队列。实际工程中可以：

  • 手写二叉堆（面试、算法题）
  • 用 d3-queue、heap-js 等第三方库
  • 数据量小时直接 sort() 也能过
