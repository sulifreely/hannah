问答 01（入门）：JS 有哪些数据类型？怎么正确判断 ​

题目 ​

请说明 JavaScript 中都有哪些数据类型，typeof、instanceof、Object.prototype.toString.call、Array.isArray 各自适合什么场景？为什么 typeof null === 'object'？new String("abc") 和 "abc" 又有什么区别？

参考答案 ​

数据类型

JavaScript 数据类型分两大类：

  • 原始类型（primitive）（7 种）：undefined、null、boolean、number、string、symbol、bigint。原始值不可变，按值传递。
  • 引用类型（object）：Object（含 Array、Function、Date、RegExp、Map、Set、Error 等），按引用传递。

四种判断方式

  1. typeof：适合判断原始类型。除了 typeof null === 'object'（历史遗留 bug，最初对象和 null 都用低三位为 0 的标签），其他原始类型都会返回自己的类型字符串；对 function 返回 'function'；对所有其他对象都返回 'object'，因此无法区分数组、日期等。
  2. instanceof：判断对象是否是某个构造函数的实例，本质是沿原型链查 constructor.prototype 是否出现过。跨 iframe/realm 时不可靠（不同 realm 的 Array 构造函数不同）。
  3. Object.prototype.toString.call(x)：万能判断，返回 "[object Xxx]" 形如 "[object Array]"、"[object Date]"、"[object Null]"。这是判断内建对象最稳的方式。
  4. Array.isArray(x)：官方专用，跨 realm 也能正确判断数组，推荐取代 instanceof Array。
js
typeof undefined              // 'undefined'
typeof 1                      // 'number'
typeof 1n                     // 'bigint'
typeof Symbol()               // 'symbol'
typeof null                   // 'object'（历史遗留）
typeof []                     // 'object'
typeof function(){}           // 'function'

[] instanceof Array           // true
[] instanceof Object          // true

Object.prototype.toString.call(null)   // '[object Null]'
Object.prototype.toString.call([])     // '[object Array]'
Object.prototype.toString.call(/a/)    // '[object RegExp]'

Array.isArray([])             // true

追问 1：为什么 typeof null === 'object'？为什么不修复？ ​

答：这是 JS 最早版本遗留的 bug。在 V8 早期实现里，类型信息存在值的低位标签中：对象的低位是 000，null 的整个值本身是全 0（表示空指针），因此低三位也是 000，typeof 把它归成了对象。后来 ECMAScript 起草时曾提议改成 typeof null === 'null'，但被否决，理由是会破坏太多存量代码——大量 if (typeof x === 'object' && x !== null) 这类判空的写法一旦语义改变就会静默出错。因此规范里明确把 typeof null === 'object' 冻结为标准行为。要判断 null，直接用 x === null 或 Object.prototype.toString.call(x) === '[object Null]'。

追问 2：new String("abc") 和 "abc" 有什么区别？ ​

答：

  • "abc" 是原始类型字符串，typeof "abc" === 'string'。
  • new String("abc") 是包装对象（Object），typeof new String("abc") === 'object'。

访问原始字符串的属性/方法（如 .length、.toUpperCase()）时，JS 会临时创建一个 String 包装对象、调用方法、再销毁，这个过程叫装箱（boxing）。所以看起来"字符串也有方法"，实则每次都是临时对象。

关键差异：

js
"abc" == new String("abc")     // true  （宽松相等会把对象拆箱）
"abc" === new String("abc")    // false （类型不同）
if (new String("")) { }        // 进入！对象永远真值
if ("") { }                    // 不进入

追问 3：判断值是不是"纯对象"（plain object），怎么写？ ​

答：所谓 plain object 通常指 {} 或 new Object() 或 Object.create(null) 生成的对象，不包括 Array、Date、Map、类实例等。可以用原型链判断：

js
function isPlainObject(v) {
  if (v === null || typeof v !== 'object') return false;
  const proto = Object.getPrototypeOf(v);
  return proto === null || proto === Object.prototype;
}

isPlainObject({});                     // true
isPlainObject(Object.create(null));    // true
isPlainObject([]);                     // false
isPlainObject(new Date());             // false
isPlainObject(new class Foo {}());     // false
