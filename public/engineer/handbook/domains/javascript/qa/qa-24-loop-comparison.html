问答 24（进阶）：遍历方式大对比：for...in vs for...of vs forEach/map/reduce ​

题目 ​

请对比 JavaScript 中常见的遍历方式：for...in、for...of、数组的 forEach/map/reduce/filter 等。它们各自遍历的是什么（键 vs 值）、适用什么场景、能否中断循环、如何处理稀疏数组？性能上有差异吗？for await...of 又是什么？

参考答案 ​

先给结论：没有最好的，只有最合适的。选哪种取决于你在遍历什么、要不要中断、要不要返回值、要不要链式调用。

一图概览 ​

方式遍历对象遍历的是能否中断返回值适用场景
for (let i = 0; i < arr.length; i++)数组/类数组索引 → 值✅ break / continue / return-需要索引、需要中断、性能敏感
for...in任何对象可枚举属性的键名（含原型链）✅ break 等-遍历对象键名（不推荐用于数组）
for...of可迭代对象（Array/Map/Set/String...）值✅ break 等-遍历值、需要中断、统一遍历语法
arr.forEach数组值 + 索引 + 数组本身❌（抛错才能停）undefined纯遍历、不关心中断
arr.map数组值 + 索引 + 数组本身❌新数组一对一映射（每个元素变一下）
arr.filter数组值 + 索引 + 数组本身❌新数组（子集）筛选符合条件的元素
arr.reduce数组值 + 索引 + 数组本身❌任意类型（累加结果）聚合/归并（把数组合成一个值）
arr.some / arr.every数组值 + 索引 + 数组本身✅（短路返回）boolean判断是否有/所有满足条件
arr.find / arr.findIndex数组值 + 索引 + 数组本身✅（找到就停）元素 / 索引找第一个符合条件的

逐个详解 ​

1. for...in —— 遍历对象的可枚举键名 ​

js
const obj = { a: 1, b: 2 };
for (const key in obj) {
  console.log(key);   // 'a', 'b'
}
1
2
3
4

特点：

  • 遍历的是键名（key），不是值。
  • 会遍历原型链上的可枚举属性——所以用 for...in 遍历对象时通常要搭配 Object.hasOwn(obj, key) 过滤。
  • 不推荐用于数组：数组也是对象，索引也是键名，但 for...in 不能保证顺序（虽然现代引擎基本按顺序，但规范不保证），而且会把你手动加到数组上的非索引属性也遍历出来。
  • 可以用 break、continue、return（在函数里）中断。

适用场景：遍历普通对象的键名、调试时查看对象所有属性。日常开发用 Object.keys/entries/values 更多，因为它们只返回自身可枚举属性，更安全。

2. for...of —— 遍历可迭代对象的值 ​

js
const arr = [10, 20, 30];
for (const val of arr) {
  console.log(val);   // 10, 20, 30
}

// 想拿索引用 entries()
for (const [i, val] of arr.entries()) {
  console.log(i, val);
}

特点：

  • 遍历的是值（value）。
  • 只要实现了 Symbol.iterator（可迭代协议）就能用：Array、String、Map、Set、arguments、NodeList、Generator……
  • 不遍历原型链，只遍历对象自身产出的值。
  • 可以用 break、continue、return、throw 中断。

适用场景：最通用的遍历语法——不确定用什么的时候用 for...of 基本不会错。尤其适合需要中断循环、或者要统一遍历多种集合类型的场景。

3. forEach —— 数组的"纯遍历" ​

js
arr.forEach((val, index, array) => {
  console.log(val, index);
});
1
2
3

特点：

  • 数组方法，回调里能拿到值、索引、数组本身。
  • 不能用 break / continue 中断。想提前停只能抛异常（不推荐）。
  • 返回 undefined。
  • 稀疏数组的空洞会被跳过（见下面追问）。

适用场景：就是想遍历每个元素做点事，不需要中断、不需要返回值。代码读起来"我就是在遍历"，语义明确。

4. map —— 一对一映射 ​

js
const doubled = arr.map(x => x * 2);
// [1, 2, 3].map(x => x * 2) → [2, 4, 6]
1
2

特点：

  • 返回一个新数组，长度和原数组一样，每个元素是回调的返回值。
  • 不能中断，每个元素都会走一遍回调。
  • 不修改原数组（除非你在回调里手动改）。
  • 稀疏数组的空洞会被保留（位置留着，不调回调）。

适用场景："每个元素做个变换，得到新数组"。React 里渲染列表就是典型的 map 场景。

⚠️ 反模式：用 map 做副作用（比如 arr.map(x => console.log(x))）——map 返回了一个你用不上的数组，浪费。纯副作用用 forEach 或 for...of。

5. filter —— 筛选 ​

js
const evens = arr.filter(x => x % 2 === 0);
1

返回满足条件的元素组成的新数组（子集，可能空）。不能中断。

6. reduce —— 归并 / 聚合 ​

js
const sum = arr.reduce((acc, cur, index, array) => acc + cur, 0);
1

特点：

  • 最强大也最灵活的数组方法，map、filter、find 理论上都能用 reduce 实现。
  • 返回值类型任意（数字、对象、数组……取决于初始值和累加逻辑）。
  • 不能中断。
  • 不传初始值时，第一次迭代 acc 是第一个元素、cur 是第二个；空数组且没初始值会抛错。推荐总是传初始值。

适用场景：求和、求最大值、数组转对象、分组、扁平化……凡是"把数组收成一个结果"的都可以用 reduce。

性能差异 ​

简单说几点结论（实际情况要看引擎、数据量、具体操作）：

  1. 普通 for 循环（带索引的）通常最快——没有函数调用开销、没有迭代器对象创建开销。大数据量下明显。
  2. for...of 略慢于普通 for——因为要创建迭代器、每次调 next()、生成 {value, done} 对象。但现代引擎优化得很好，差异不大（除非数据量特别大）。
  3. forEach / map 比 for...of 慢一点——多了函数调用开销（每次回调的入栈出栈），但引擎也会做内联优化。
  4. reduce 一般比 map + filter 链式调用快——只遍历一次，而链式要遍历多次。但可读性差一些。
  5. for...in 最慢——因为要查原型链、还要做属性枚举，开销大。千万别用它遍历数组。

实际建议：

  • 数据量小（几千以内）：怎么写都行，可读性优先。
  • 数据量大（几万+）且在热点路径：再考虑性能，用普通 for 或 for...of。
  • 不要过早优化——先写对，再用 Profiler 测，发现瓶颈再优化。

稀疏数组处理 ​

稀疏数组就是"有洞"的数组，比如 Array(5)、[1, , 3]。

方式对稀疏数组的行为
普通 for 循环每个索引都访问，空洞处是 undefined
for...of每个索引都访问，空洞处是 undefined
forEach跳过空洞，不调用回调
map跳过空洞，但空洞位置保留在结果数组里
filter跳过空洞，结果数组里没有空洞位置
reduce跳过空洞
for...in跳过空洞（因为没有这个属性键）
js
const sparse = [1, , 3];   // length 是 3，索引 1 是空的

for (let i = 0; i < sparse.length; i++) {
  console.log(i, sparse[i]);   // 0 1, 1 undefined, 2 3
}

sparse.forEach((v, i) => console.log(i, v));   // 0 1, 2 3  （跳过了 1）

sparse.map(x => x * 2);   // [2, 空, 6]  （空洞保留）

为什么 forEach 等要跳过空洞？因为规范定义它们是"按枚举属性来的"，稀疏数组的空洞位置没有对应属性（虽然 arr[1] 读出来是 undefined，但那是原型链查找的结果，不是自身有这个属性）。

日常开发尽量别用稀疏数组——容易出 bug。

追问 1：for...in 和 for...of 有什么本质区别？分别用在什么场景？ ​

答 ​

本质区别一句话：for...in 遍历对象的可枚举键名（属性名），for...of 遍历可迭代对象的值。

详细对比：

对比项for...infor...of
遍历内容键名（key / property name）值（value）
遍历对象任何对象（Object）实现了 Symbol.iterator 的可迭代对象
原型链会遍历原型链上的可枚举属性不涉及原型链，由迭代器决定产出什么
数组使用不推荐（顺序不保证、会遍历自定义属性）推荐，直接拿值
可中断✅ break / continue✅ break / continue
引入版本ES5 之前就有ES6
典型场景遍历普通对象的键遍历数组/Map/Set/Generator 等的值

for...in 用在对象上：

js
const obj = { a: 1, b: 2 };
for (const key in obj) {
  if (Object.hasOwn(obj, key)) {   // 过滤掉原型链上的
    console.log(key, obj[key]);
  }
}

但现在更推荐 Object.keys(obj).forEach(...) 或 for (const [k, v] of Object.entries(obj))，更直观，也自动过滤了原型链。

for...of 用在可迭代对象上：

js
// 数组
for (const val of [1, 2, 3]) { /* ... */ }

// Map：直接解构 key value
for (const [k, v] of new Map([['a', 1]])) { /* ... */ }

// 字符串（按 Unicode 码点）
for (const ch of 'hello') { /* ... */ }

追问 2：map 和 forEach 除了返回值，还有什么区别？各自在什么场景用？ ​

答 ​

核心区别 ​

对比项forEachmap
返回值undefined新数组（和原数组同长度）
语义"遍历每个元素做点事"（副作用）"把每个元素映射成新元素"（纯转换）
链式调用不能（返回 undefined）能（返回数组，可以继续 .filter().reduce()...）
稀疏数组跳过空洞跳过空洞但保留位置
性能稍快一点（不用分配新数组）稍慢（分配新数组内存）

适用场景 ​

forEach 适合：

  • 就是要遍历做副作用（比如 console.log、发请求、修改外部变量、操作 DOM）。
  • 不需要返回值。
js

// 操作 DOM
buttons.forEach(btn => btn.addEventListener('click', handler));

追问 3：for await...of 是什么？和同步的 for...of 有什么区别？举个实际应用场景。 ​

答 ​

for await...of 是 ES2018 引入的异步迭代语法，专门用来遍历异步可迭代对象（Async Iterable）——也就是每次产出的值都是异步的（Promise），需要等一个完成了再拿一下。

什么是异步可迭代 ​

同步可迭代对象有 Symbol.iterator，返回迭代器，迭代器的 next() 返回 { value, done }。

异步可迭代对象有 Symbol.asyncIterator，返回异步迭代器，异步迭代器的 next() 返回 Promise<{ value, done }>。

js
const asyncIterable = {
  [Symbol.asyncIterator]() {
    let i = 0;
    return {
      async next() {
        if (i < 3) {
          await new Promise(r => setTimeout(r, 500));   // 模拟异步
          return { value: i++, done: false };
        }
        return { done: true };
      },
    };
  },
};

和同步 for...of 的区别 ​

对比项for...offor await...of
迭代协议Symbol.iterator（同步迭代器）Symbol.asyncIterator（异步迭代器）
next() 返回{ value, done }Promise<{ value, done }>
所在函数普通函数即可必须在 async function 里
每次循环同步执行自动 await 下一个值
错误处理try/catch 捕获同步错误try/catch 捕获 reject

异步 Generator ​

async function* 定义的异步 Generator 自动实现了 Symbol.asyncIterator，写起来最方便：

js
async function* fetchPages(pageIds) {
  for (const id of pageIds) {
    const res = await fetch(`/api/page/${id}`);
    const data = await res.json();
    yield data;
  }
}

// 消费
async function showAll() {
  for await (const page of fetchPages([1, 2, 3])) {
    render(page);
  }
}

实际应用场景 ​

1. Node.js 可读流（Readable Stream）逐块读取

js
const fs = require('fs');
async function readFileLineByLine(path) {
  const stream = fs.createReadStream(path, { encoding: 'utf8' });
  let rest = '';
  for await (const chunk of stream) {    // Node 10+ 支持
    const lines = (rest + chunk).split('\n');
    rest = lines.pop();
    for (const line of lines) {
      console.log('一行:', line);
    }
  }
}

async function* loadAllUsers() {
  let cursor = null;
  while (true) {
    const { data, nextCursor } = await fetchUsers({ cursor });
    yield* data;
    if (!nextCursor) break;
    cursor = nextCursor;
  }
}
