问答 22（进阶）：Iterator 和 Generator（迭代器与生成器） ​

题目 ​

请说明迭代器协议（Iterator Protocol）和可迭代协议（Iterable Protocol）。for...of 的原理是什么？Generator 函数有哪些语法特性？yield、next、return、throw 分别怎么用？异步迭代（for await...of）又是什么？

参考答案 ​

为什么需要迭代器协议 ​

JS 里有很多"可以遍历的东西"：数组、字符串、Map、Set、arguments、NodeList……它们结构不一样，但都想被统一地遍历。

在 ES6 之前，for 循环、forEach、for...in 各有各的用法，没有统一的"遍历"抽象。迭代器协议就是为了解决这个问题——只要你遵守这个协议，就能被 for...of、解构赋值、扩展运算符、Array.from 等统一消费。

两个核心协议 ​

1. 迭代器协议（Iterator Protocol） ​

一个对象只要有 next() 方法，每次调用 next() 返回 { value, done }，它就是一个迭代器。

  • value：当前迭代的值
  • done：布尔值，true 表示迭代结束
js
const iterator = {
  count: 0,
  next() {
    return {
      value: this.count++,
      done: this.count > 3,
    };
  },
};

iterator.next();   // { value: 0, done: false }
iterator.next();   // { value: 1, done: false }
iterator.next();   // { value: 2, done: false }
iterator.next();   // { value: undefined, done: true }

2. 可迭代协议（Iterable Protocol） ​

一个对象只要有 Symbol.iterator 方法（键是 Symbol.iterator），这个方法返回一个迭代器，它就是可迭代对象。

js
const range = {
  from: 1,
  to: 5,
  [Symbol.iterator]() {
    // 返回一个迭代器对象
    return {
      current: this.from,
      last: this.to,
      next() {
        if (this.current <= this.last) {
          return { value: this.current++, done: false };
        } else {
          return { value: undefined, done: true };
        }
      },
    };
  },
};

// 现在 range 可以被 for...of 遍历了
for (const num of range) {
  console.log(num);   // 1, 2, 3, 4, 5
}

for...of 的原理 ​

for...of 自动做了三件事：

  1. 调用可迭代对象的 [Symbol.iterator]()，拿到迭代器。
  2. 循环调用迭代器的 .next()，把 value 赋给循环变量。
  3. 直到 done === true 停止。

伪代码：

js
const iterator = iterable[Symbol.iterator]();
let result;
while (!(result = iterator.next()).done) {
  const item = result.value;
  // ... 循环体
}

所以：只要实现了可迭代协议，就能被 for...of 遍历。

内置的可迭代对象 ​

以下对象天然有 Symbol.iterator：

  • Array
  • String（按 Unicode 码点遍历）
  • Map（遍历 [key, value]）
  • Set
  • arguments
  • NodeList、HTMLCollection 等 DOM 集合
  • TypedArray

普通对象 默认不是可迭代的——因为对象的遍历顺序语义不明确（ES5 之前甚至没规定顺序），而且 for...in 已经能遍历键了。

其他消费可迭代对象的语法 ​

不只是 for...of，这些也都是消费可迭代对象：

js
[...range];                    // 扩展运算符
const [a, b] = range;          // 解构赋值
Array.from(range);             // Array.from
new Set(range);                // Set 构造函数
new Map(entries);              // Map 构造函数
Promise.all(iterable);         // 接受可迭代的 Promise
yield* iterable;               // Generator 里的 yield*

Generator 函数 ​

Generator（生成器）是一种特殊的函数，它可以暂停执行、稍后恢复。它既是迭代器（有 next 方法），又是可迭代对象（有 Symbol.iterator），天生就能被 for...of 消费。

语法 ​

js
function* gen() {
  yield 1;
  yield 2;
  yield 3;
}

const g = gen();   // 调用 Generator 函数不执行函数体，返回一个迭代器对象

g.next();          // { value: 1, done: false }
g.next();          // { value: 2, done: false }
g.next();          // { value: 3, done: false }
g.next();          // { value: undefined, done: true }

关键点：

  • 函数名前面加 *（function* 或 function *，空格无所谓，推荐 function*）。
  • 函数体里用 yield 产出值。
  • 调用 gen() 不执行函数体，返回一个迭代器对象。
  • 每次调用 .next()，函数从上次暂停的地方继续执行，直到遇到下一个 yield 或函数结束。

yield 是双向通道 ​

yield 不只是"产出值"，还能接收外部传进来的值：

js
function* gen() {
  const x = yield 'a';   // 第一个 yield 产出 'a'，下一次 next(v) 的 v 赋给 x
  console.log('x =', x);
  const y = yield 'b';
  console.log('y =', y);
}

const g = gen();
g.next();          // { value: 'a', done: false }  （第一次 next 的参数会被忽略）
g.next(100);       // x = 100 → { value: 'b', done: false }
g.next(200);       // y = 200 → { value: undefined, done: true }

yield* —— 委托迭代 ​

yield* 后面跟一个可迭代对象，相当于把它展开，逐个 yield：

js
function* inner() {
  yield 'a';
  yield 'b';
}

function* outer() {
  yield 'start';
  yield* inner();        // 委托给 inner
  yield* [1, 2, 3];     // 也可以委托给数组（可迭代对象就行）
  yield 'end';
}

[...outer()];   // ['start', 'a', 'b', 1, 2, 3, 'end']

异步迭代与 for await...of ​

ES2018 引入了异步迭代器和 for await...of，专门对付"异步产出的数据"——比如逐块读取的文件流、分页请求的数据、WebSocket 消息流。

异步迭代器协议 ​

和同步迭代器类似，但 next() 返回的是 Promise：

js
const asyncIterator = {
  async next() {
    // ...异步操作
    return { value: ..., done: ... };

};

异步可迭代协议 ​

const asyncRange = {

  to: 3,

    return {
      current: this.from,
      last: this.to,
      async next() {
        await new Promise(r => setTimeout(r, 500));   // 模拟异步
        if (this.current <= this.last) {
          return { value: this.current++, done: false };
        }
        return { done: true };

};

追问 1：对象（plain object）为什么默认不能用 for...of 遍历？怎样让它可以？ ​

答 ​

为什么默认不行 ​

两个原因：

  1. 历史原因：ES5 及以前，对象属性的遍历顺序没有规范。不同引擎顺序可能不一样，用 for...of 这种"按顺序遍历"的语义会有歧义。
  2. 语义原因：对象是"键值对"的集合，不是"序列"。数组是有序序列，用 for...of 遍历值很自然；对象遍历什么——键、值、还是键值对？不明确。for...in 已经负责遍历键（还包括原型链上的），for...of 就不默认支持了。

怎么让对象可以 for...of ​

给它实现 Symbol.iterator 就行。三种常见方式：

方式一：自定义迭代器（遍历值）

js
const obj = { a: 1, b: 2, c: 3 };
obj[Symbol.iterator] = function* () {
  for (const key of Object.keys(this)) {
    yield this[key];
  }
};

for (const v of obj) {
  console.log(v);   // 1, 2, 3
}

js

方式三：用 Map 代替对象

Map 天生就是可迭代的，而且键可以是任意类型、保证插入顺序：

js
const map = new Map([['a', 1], ['b', 2]]);
for (const [k, v] of map) { /* ... */ }
for (const k of map.keys()) { /* ... */ }
for (const v of map.values()) { /* ... */ }

追问 2：Generator 和 async/await 有什么关系？async/await 是不是 Generator 的语法糖？ ​

答 ​

可以说 async/await 就是 Generator + Promise 的语法糖，专门针对异步场景做了优化。

回顾 Generator 怎么处理异步 ​

在没有 async/await 的年代，人们用 Generator + 自动执行器（co 库）来写"看起来同步"的异步代码：

js
// 用 Generator 写异步
function* fetchData() {
  const user = yield fetch('/user');
  const posts = yield fetch(`/posts?userId=${user.id}`);
  return posts;
}

// 需要一个 co 自动执行器来驱动
co(fetchData).then(posts => console.log(posts));

  const user = await fetch('/user');
  const posts = await fetch(`/posts?userId=${user.id}`);
  return posts;

fetchData().then(posts => console.log(posts));

但两者不完全等价 ​

  • async/await 专门为异步优化：返回值一定是 Promise、await 后面一定是 Promise（非 Promise 也会被包装）、错误自动 reject。
  • Generator 更通用：不止用于异步，还能用来实现惰性序列、状态机、协程等。
  • 错误处理：async 函数里抛错会导致返回的 Promise reject；Generator 里抛错需要调用者用 .throw() 传入或用 try/catch 包裹。

所以说 async/await 是 "专门用于异步场景的 Generator 语法糖 + 内置执行器"，理解 Generator 有助于理解 async/await 底层是怎么跑的。

追问 3：Generator 除了迭代数据，还有哪些实际应用场景？ ​

答 ​

Generator 的本质是可暂停、可恢复的函数，这个特性让它不止能用来产出序列，还有很多巧妙的用法。

1. 惰性求值 / 无限序列 ​

js
function* fibonacci() {
  let a = 0, b = 1;
  while (true) {
    yield a;
    [a, b] = [b, a + b];
  }
}

const fib = fibonacci();
fib.next().value;   // 0
fib.next().value;   // 1
fib.next().value;   // 1
fib.next().value;   // 2
// 想用多少取多少，不用一次生成所有

4. 实现可取消的任务 ​

因为可以暂停，所以可以"随时中断"：

js
function* longTask() {
  for (let i = 0; i < 1e6; i++) {
    // 每 1000 次让出控制权
    if (i % 1000 === 0) yield i;
    // ... 计算
  }
}

const task = longTask();
// 想停就停（不调 next 了就行）
