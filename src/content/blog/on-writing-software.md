软件的本质是管理复杂度。我们写下的每一行代码，都是在与未来的自己和同事沟通。

## 简单优先

> 简单是可靠的前提。

能用一个函数解决的，不要引入一个框架。能用约定解决的，不要引入配置。复杂度一旦引入就很难移除。

## 为变化设计

```ts
// 把易变的部分隔离在边界处
interface Storage {
  get(key: string): Promise<string | null>;
  set(key: string, value: string): Promise<void>;
}
```

把稳定的核心和易变的细节分开，是大多数好架构的共同点。

## 保持克制

抵御过度设计的诱惑。先让它跑起来，再让它对，最后才让它快。
